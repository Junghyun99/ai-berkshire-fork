## 왜 AI에게 그냥 물으면 안 되나?

물론 Claude에게 바로 물을 수 있다: "핀둬둬 살 만한지 분석해줘". 그러면 "한편으로는... 다른 한편으로는..." 식의 균형 잡힌 분석을 받고, "투자에는 위험이 따르니 스스로 판단하세요"로 끝난다.

**이런 분석은 맞아 보이지만, 의사결정에 쓸 수 없다.**

AI Berkshire가 푸는 문제는 "분석이 가능한가"가 아니라 **분석의 품질과 의사결정의 규율**이다. 핵심 차이:

### 1. 결론을 강제한다, 얼버무리지 않는다

AI에게 그냥 물으면 양다리 걸치는 "분석"을 받는다. AI Berkshire는 강제로 출력한다: **통과/미통과/회색 지대**, 구체적 가격 구간과 유형별 제안 포함.

> 일반 AI의 답: *"핀둬둬는 성장 잠재력이 있으나 경쟁 압박도 있어 투자자는 저울질이 필요합니다..."*
>
> AI Berkshire의 출력:

> | 전략 | 제안 | 가격 구간 |
> |------|------|---------|
> | 공격형 | 현재 가격에서 20% 비중 구축 가능 | $95-105 |
> | 균형형 | 자사주 매입 정책 명확해진 후 진입 | $85-95 |
> | 보수형 | 10년 확실성 기준 미달, 관망 | — |
>
> **거울 테스트**: 5문장으로 못 쓰면 = 사지 않는다. 예외 없다.

### 2. 4대 거장 관점의 대립, 단일 분석이 아니다

"버핏 방식으로 분석해줘" 수준이 아니다. 네 관점은 **진짜 모순과 긴장**을 만들어낸다 —

핀둬둬를 예로:
- **돤융핑** (비즈니스 모델): 좋은 사업, C2M 모델은 복제 어려움 → 점수 3.7/5
- **버핏** (재무 밸류에이션): 현금 차감 PER 겨우 6.3x, 현금 인쇄기 → 점수 4.4/5
- **멍거** (역발상): 해자가 생각보다 얕다. 더우인이 3년 만에 GMV 4조를 만들었다 → 점수 3.5/5
- **리루** (장기 확실성): 경영진 문화에 잠재 리스크, 10년 후 불확실 → 점수 2.0/5

**버핏은 "진짜 싸다"는데 리루는 "불확실하면 안 산다"** — 이런 충돌이야말로 투자 의사결정의 실제 모습이다. 단일 prompt로는 이런 다관점 대립을 만들 수 없고, 바로 이것이 맹점을 피하는 열쇠다.

### 3. 구조화된 반편향 메커니즘

AI의 가장 큰 위험은 틀린 답이 아니라 **맞아 보이지만 검증을 못 버티는** 답이다. AI Berkshire는 프로세스에 다층 "사기 방지" 메커니즘을 내장했다:

| 메커니즘 | 해결하는 문제 | 예시 |
|------|------------|------|
| **정보 풍부도 등급 (A/B/C)** | "자료 많음 = 확실성 높음"이라는 환상 방지 | 팝마트는 B등급: 데이터 제한적, 추산 지표에 신뢰도 표기 |
| **멍거식 역방향 검증** | 실패 시나리오 사고 강제 | "어떤 경우에 핀둬둬가 죽는가?" → 5대 시나리오와 확률 나열 |
| **빠른 부결 리스트** | 8개 레드라인 단독 부결 | 경영진 정직성 오점 → 밸류에이션이 아무리 싸도 바로 부결 |
| **반컨센서스 점검** | 시장과 같은 생각 방지 | "똑똑한 사람들은 왜 공매도 중인가?" → 간과된 리스크 발견 |
| **여백의 원칙** | 차라리 "모른다"고 말한다 | 데이터 부족 시 "회색 지대" 표기, 추측으로 확실성을 가장하지 않음 |

### 4. 금융 데이터의 정밀성

LLM의 암산은 믿을 수 없다. PER의 소수점 하나, 시총 단위에서 홍콩달러와 위안을 혼동하면 잘못된 투자 결정으로 이어질 수 있다.

**실제 사례**: 텐센트 분석 시 출처마다 시총 단위가 "홍콩달러 억"과 "위안 억"으로 달랐다. AI Berkshire의 처리 방식:

```bash
# 시총 수동 검산: 주가 × 총주식수, 보고된 데이터와 대조
python3 tools/financial_rigor.py verify-market-cap \
  --price 510 --shares 9.11e9 --reported 4.65e12 --currency HKD
# ✅ 검증 통과, 편차 0.08%
```

모든 계산은 Python `decimal.Decimal`(정밀 십진수)을 사용하고 `float`를 쓰지 않는다. 핵심 데이터는 최소 2개의 독립 출처로 교차 검증한다.

### 5. 재현 가능한 리서치 프로세스

AI에게 그냥 물으면 매번 출력의 형식, 깊이, 커버리지가 다르다 — 오늘 텐센트 분석에는 해자 점수가 있는데, 내일 메이투안 분석에서는 빠질 수 있다.

AI Berkshire는 보장한다: **같은 입력 → 구조가 일치하고 깊이가 일치하는 출력**. 이것이 의미하는 바:
- 7개 기업 횡단 비교, 채점 기준 완전 일치
- 같은 회사를 반년 후 재분석하면 변화를 바로 대조 가능
- 팀원 간 리서치 결과를 정렬 가능

> 실제 출력 — 7개 기업을 같은 기준의 Checklist로 스크리닝:
>
> | 회사 | 통과? | 능력범위 | 좋은사업 | 해자 | 경영진 | 안전마진 | 종합 |
> |------|:-----:|:------:|:------:|:------:|:------:|:-------:|:----:|
> | 마오타이 | ✅ 통과 | ★★★★★ | ★★★★★ | ★★★★★ | ★★★☆☆ | ★★★★☆ | 4.7 |
> | 텐센트 | ✅ 통과 | ★★★★☆ | ★★★★★ | ★★★★★ | ★★★★★ | ★★★★☆ | 4.7 |
> | 엔비디아 | ✅ 조건부 | ★★★★☆ | ★★★★★ | ★★★★★ | ★★★★★ | ★★★☆☆ | 4.3 |
> | 메이투안 | ✅ 조건부 | ★★★★☆ | ★★★★☆ | ★★★★☆ | ★★★★☆ | ★★★★☆ | 4.0 |
> | 콰이쇼우 | ✅ 조건부 | ★★★☆☆ | ★★★★☆ | ★★★★☆ | ★★★★☆ | ★★★★★ | 4.0 |
> | 핀둬둬 | ❓ 회색 | ★★★★☆ | ★★★★☆ | ★★★☆☆ | ★★★☆☆ | ★★★★★ | 3.8 |
> | 팝마트 | ❓ 회색 | ★★★☆☆ | ★★★★☆ | ★★★★☆ | ★★★★★ | ★★★☆☆ | 3.7 |

### 6. 멀티 Agent 병렬 = 리서치 깊이의 배가

`/investment-team`은 4개의 독립 Agent를 **동시에** 실행해 한 회사를 연구한다. 각 Agent가 각자 웹을 검색하고, 데이터를 교차 검증하고, 독립적으로 결론을 낸다. 하나의 prompt를 네 조각으로 나눈 게 아니다 — 4명의 "애널리스트"가 각자 완전한 리서치를 하고 Team Lead가 종합한다.

혼자 AI에게 물으면 컨텍스트 창은 하나다. 4개 Agent 병렬은 4배의 검색량, 4배의 정보원, 4개의 독립 관점을 의미한다.

```
┌─────────────────────────────────────────────┐
│              Team Lead (당신)                │
│         총괄 조율 · 종합 판단                 │
├──────┬──────┬──────────┬───────────┤
│ Agent 1    │ Agent 2    │ Agent 3        │ Agent 4         │
│ 비즈니스모델│ 재무밸류    │ 산업경쟁       │ 리스크·경영진    │
│ 돤융핑관점  │ 버핏관점    │ 멍거관점       │ 리루관점         │
└──────┴──────┴──────────┴───────────┘
        ↓ 병렬 리서치, 실시간 진행 보고 ↓
              최종 종합 보고서
```

### 한 문장 요약

> **AI에게 그냥 물으면 "맞아 보이는 분석"을 받고, AI Berkshire를 쓰면 "의사결정에 쓸 수 있는 투자 리서치 보고서"를 받는다.**

## 빠른 시작

### 1. AI 클라이언트 설치

이 저장소는 하나의 canonical workflow를 유지하면서 Claude Code commands와 Codex skills를 각각 제공한다. 사용하는 클라이언트에 맞게 설치하면 된다.

Claude Code 사용자:

```bash
npm install -g @anthropic-ai/claude-code
```

Codex 사용자:

```bash
# macOS / Linux
curl -fsSL https://chatgpt.com/codex/install.sh | sh

# 또는 npm
npm install -g @openai/codex

# 또는 Homebrew
brew install --cask codex

# 설치 확인
codex --version
```

Windows 사용자는 공식 PowerShell 설치 명령을 쓸 수 있다: `powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"`

`codex --version`이 정상적으로 버전을 출력하면 이 프로젝트의 Codex skills 설치를 계속할 수 있다.

#### 권한 확인 줄이기

이 skills는 도구를 빈번하게 호출하는데, Claude Code는 기본적으로 매번 권한 확인을 요청한다. 이는 Claude Code 클라이언트의 권한 메커니즘이지 이 저장소가 바꿀 수 있는 기본값이 아니다.

현재 workflow를 신뢰하고 신뢰할 수 있는 환경에서 실행 중이라면, Claude Code의 권한 확인 생략 모드로 시작할 수 있다:

```bash
claude --dangerously-skip-permissions
```

주의: 이 모드는 Claude Code의 도구 승인 보호를 끈다. 저장소, 명령, 작업 디렉토리를 신뢰하는 경우에만 사용하라.

### 2. Skills 설치

Claude Code 사용자:

```bash
# 저장소 클론
git clone https://github.com/Junghyun99/ai-berkshire-fork.git ai-berkshire

# skills를 Claude Code 전역 commands 디렉토리에 복사
cd ai-berkshire
./scripts/install-claude-commands.sh
```

Codex 사용자:

```bash
# 저장소 클론
git clone https://github.com/Junghyun99/ai-berkshire-fork.git ai-berkshire

# Codex skills를 생성해 ~/.codex/skills 에 설치
cd ai-berkshire
./scripts/install-codex-skills.sh

# 선택: Codex slash prompts를 ~/.codex/prompts 에 설치
# Claude Code의 /investment-research 에 가까운 경험을 위해
./scripts/install-codex-prompts.sh
```

저장소는 세 가지 입구를 함께 유지한다: `skills/*.md`는 Claude Code command 소스 파일; `codex-skills/*/SKILL.md`는 `scripts/sync-codex-skills.py`가 `skills/*.md`에서 생성하는 Codex skill 패키지; `codex-prompts/*.md`는 선택적인 Codex slash prompt 호환 계층.

Claude Code에서 바로 호출:

```bash

# 산업 스크리닝

/bottleneck-hunter AI인프라

/news-pulse 텐센트

# 사고 도구
/dyp-ask 핀둬둬의 해자는 도대체 어디에 있나?
/wechat-article 메이투안
```

Codex에서는 설치 후 Codex를 재시작하고 skill 이름으로 작업을 서술하면 된다. 예:

```text
investment-research 로 텐센트 연구
earnings-review 로 PDD 2025 연간보고서 분석

wechat-article 로 메이투안 투자 아티클 작성
```

Codex slash prompts를 설치했다면 재시작 후 `/` 메뉴에서 검색할 수 있다. Codex 공식 custom prompt 입구는 보통 `prompts:<name>`으로 표시된다. 예:

```text
/prompts:investment-research 텐센트
```

## 각 Skill 상세 소개

### 1. `/investment-research` — 4대 거장 종합 분석

가장 포괄적인 단일 기업 심층 리서치 프레임워크. 7개 모듈을 순서대로 실행한다:

```
데이터 수집 → 사업 본질(돤융핑) → 해자(버핏) → 역발상(멍거)
    → 경영진 평가(돤융핑+버핏) → 문명 트렌드(리루) → 밸류에이션과 안전마진
```

**핵심 특징**:
- AI 리서치 편향 자각 메커니즘 (A/B/C 정보 풍부도 등급)
- 핵심 데이터 다중 출처 교차 검증 (시총 수동 검산, 최소 2개 독립 출처)
- 네 거장의 "되물음"이 전체를 관통
- 3개 시나리오 밸류에이션 (낙관/중립/비관) + 역방향 DCF

**출력 예시 발췌**:

> #### 종합 의사결정 메모
>
> | 차원 | 결론 | 신뢰도 |
> |------|------|--------|
> | 사업 품질 (돤융핑) | 탁월: 플랫폼형 사업, 양면 네트워크 효과, 한계비용 0에 수렴 | ★★★★★ |
> | 해자 (버핏) | 넓고 넓어지는 중: 네트워크 효과+전환비용+규모의 경제 삼중 중첩 | ★★★★☆ |
> | 경영진 (돤융핑+버핏) | 우수: 창업자가 키를 잡고, 자본 배분 규율 강함 | ★★★★☆ |
> | 최대 리스크 (멍거) | 규제 정책 불확실성, 신사업 적자가 전체 이익을 잠식 | ★★★☆☆ |
> | 문명 트렌드 (리루) | 디지털 소비 트렌드에 부합하나 "문명급 패러다임 전환"은 아님 | ★★★★☆ |
> | 밸류에이션 (버핏+돤융핑) | 현재 PER 18x, 역사적 중위값 대비 낮은 편, 일정한 안전마진 | ★★★★☆ |
>
> **돤융핑**: "이 사업의 본질은 소비자와 상인을 연결하는 것이고, 버는 돈은 효율 향상의 돈이다. 좋은 사업의 표시: 사용자가 많을수록 상인이 많아지고, 상인이 많을수록 사용자가 많아진다. 플라이휠이 한번 돌기 시작하면 멈추기 어렵다."
>
> **멍거**: "거꾸로 생각해보라 — 이 회사가 내일 사라지면 사용자와 상인은 어떻게 될까? 답이 '금방 대체재를 찾는다'면 해자가 충분히 깊지 않은 것이다. 답이 '생활이 몹시 불편해진다'면 주목할 가치가 있다."

### 2. `/investment-team` — 멀티 Agent 투자 리서치 팀

4개의 AI Agent를 병렬 실행해 실제 투자 리서치 팀의 협업을 시뮬레이션한다. 각 Agent가 독립 검색, 독립 분석, 독립 채점하고 마지막에 Team Lead가 종합 판단한다.

**출력 예시 발췌**:

> #### 한 문장 결론
> 메이투안은 중국 로컬 생활 서비스의 절대적 선두로, 다중 네트워크 효과 해자를 보유하며, 현재 밸류에이션은 역사적 저점 수준으로 장기 투자 가치가 뚜렷하다. 저가 매수 권장.
>
> #### 4차원 점수 총괄표
>
> | 차원 | 프레임워크 | 점수 | 핵심 판단 |
> |------|------|------|----------|
> | 비즈니스 모델 & 해자 | 돤융핑 | ★★★★☆ | 양면 네트워크 효과 강력, 배달+매장방문이 플라이휠 형성 |
> | 재무 & 밸류에이션 | 버핏 | ★★★★☆ | 핵심 사업 이익률 지속 개선, 밸류에이션 역사적 저점 |
> | 산업 & 경쟁 | 멍거 | ★★★☆☆ | 더우인의 매장방문 사업 침공, 경쟁 구도 악화 리스크 |
> | 리스크 & 경영진 | 리루 | ★★★★☆ | 왕싱의 전략적 안목 탁월, 다만 신사업의 자금 소진 경계 |
>
> **종합 점수: 3.8 / 5**
>
> #### 투자 제안
>
> | 전략 | 제안 | 가격 구간(홍콩달러) |
> |------|------|---------------|
> | 공격형 | 현재 가격에서 30% 비중 구축 가능 | 120-140 |
> | 균형형 | 100-110까지 조정 시 진입 | 100-120 |
> | 보수형 | 분기 실적으로 이익률 추세 확인 후 진입 | <100 |

### 3. `/investment-checklist` — 버핏 매수 전 Checklist

6관문 빠른 스크리닝으로 10분 안에 이 회사가 깊이 연구할 가치가 있는지 결정한다:

```
제1관문: 능력의 범위 (내가 이해할 수 있나?)
    ↓ 통과
제2관문: 좋은 사업 (경제적 특성은?)
    ↓ 통과
제3관문: 해자 (경쟁 우위가 깊은가?)
    ↓ 통과
제4관문: 경영진 (신뢰할 수 있나?)
    ↓ 통과
제5관문: 안전마진 (가격이 싼가?)
    ↓ 통과
제6관문: 의사결정 규율 (이성인가 FOMO인가?)
    ↓ 통과
   ✅ 거울 테스트
```

**다중 기업 비교 지원** — 한 번에 여러 종목 스크리닝:

```
/investment-checklist 텐센트, 알리바바, 메이투안, 핀둬둬
```

**출력 예시 발췌**:

> #### 거울 테스트
>
> "나는 380홍콩달러에 텐센트를 매수한다. 이유는:
> 1. 이 사업의 본질은 **소셜 네트워크 + 디지털 콘텐츠 플랫폼**이고, 나는 그것을 이해한다;
> 2. 해자는 **12억 사용자의 소셜 관계망**이며, 넓어지고 있다;
> 3. 경영진은 **포니 마가 저자세로 실용적이고 자본 배분이 우수**해 신뢰할 만하다;
> 4. 현재 가격은 내재가치의 **80%** 수준으로 일정한 안전마진이 있다;
> 5. 내가 틀리더라도 하방 리스크는 통제 가능하다. **장부상 순현금 2,000억 초과, 게임 현금흐름 강력**하기 때문이다."
>
> ✅ 거울 테스트 통과
>
> **5문장으로 못 쓰면 = 사지 않는다. 예외 없다.**

### 4. `/industry-research` — 밸류체인 전경 스캔

하나의 투자 테마에서 출발해 밸류체인 전경 연구를 완성한다:

```
투자 논리 사슬 구축 → 밸류체인 전경도 → 글로벌 상장사 스캔
    → 단계별 선두 기업 4대 거장 분석 → 포트폴리오 배분 제안
```

**출력 예시 발췌**:

> #### 투자 논리 사슬: 원전
>
> 저변 트렌드: AI 데이터센터 전력 수요 폭발 + 탄소중립 목표
> → 유발: 안정적 청정 기저 전원 수요 급증
> → 창출: 원전 재가동/신규 건설/SMR의 확실한 수요
> → 수혜: 우라늄 광산 → 연료 가공 → 장비 제조 → 운영사
>
> #### 추천 포트폴리오
>
> | 계층 | 비중 | 종목 | 단계 | 핵심 논리 |
> |------|------|------|------|---------|
> | 핵심 | 50% | CGN, Cameco | 운영+우라늄 | 확실성 최고 |
> | 위성 | 30% | CNNC, 둥팡전기 | 운영+장비 | 국산 대체 수혜 |
> | 옵션 | 15% | NuScale, Nano Nuclear | SMR | 고위험 고탄력 |
> | ETF | 대체 | URA, URNM | 전체 체인 | 게으른 방안 |

### 5. `/industry-funnel` — 산업 퍼널 스크리닝

하나의 산업/방향에서 출발해 **전체 시장 → ≤10개 → 3개**로 층층이 정선한다:

```
전체 시장 스캔 (활발도 + 상승률 + 시총 상위 30 합집합, 30-60개)
    ↓ 가치투자 5대 정량 지표
1차 스크리닝 ≤ 10개
    ↓ 정밀 분석 (기업당 300-500자)
정밀 분석 ≤ 10개
    ↓ 최종 선정 (포트폴리오 상호보완성 기준, 점수 상위 3이 아님)
4대 거장 심층 분석 3개 (기업당 800-1200자)
    ↓
추천 포트폴리오 (핵심 / 위성 / 옵션) + 매매 신호
```

**핵심 특징**:
- 각 층에 명확한 잔류/탈락 기준, 탈락 종목에는 탈락 이유 기록 (블랙박스 아님)
- 최종 3개는 "포트폴리오 상호보완성"으로 선정 (고확실성 + 중간 탄력 + 고탄력), 점수 상위 3 정렬이 아님
- "미래 IPO 후보" 강제 나열로 1차 시장의 핵심 플레이어 누락 방지
- AI 편향 자각 메커니즘: 선두 기업 선호 / 영어 선호 / 스토리 선호 / 상장사 선호 대응

**`/industry-research`와의 차이**:
- `industry-research`는 밸류체인 구조와 전경에 중점 (단계별 절단)
- `industry-funnel`은 개별 종목 스크리닝 퍼널에 중점 (전체 시장에서 3개까지 정선)

**실측: AI 산업 4개 세부 분야 병렬 (2026-05-09)**:

| 세부 분야 | 최종 3개 | 핵심 비중 추천 |
|-------|---------|------------|
| AI 컴퓨팅 | TSMC / NVIDIA / SK하이닉스 | TSMC ★★★★★ |
| AI 모델 | Alphabet / Meta / 알리바바 | Alphabet ★★★★★ |
| AI 응용 | 마이크로소프트 / Adobe / AppLovin | 마이크로소프트 + Adobe ★★★★ |
| AI 인프라 전력 | Eaton / TBEA / Talen Energy | Eaton + TBEA ★★★★ |

**핵심 발견**: AI 응용층 최대 승자는 AI 네이티브 기업이 아니라 채널+데이터+워크플로 침투도를 가진 성숙한 거인들이다 — 이는 1995-2000 인터넷 버블의 "삽 파는 자" 역사 법칙과 호응한다 (아마존과 애플이 이기고 Pets.com이 졌다).

### 6. `/private-company-research` — 비상장 기업 심층 연구

정보가 희소한 비상장 기업 전용으로 설계된 "탐정식" 리서치 프레임워크:

**핵심 차별화**:
- **재무 데이터 짜맞추기**: 상장 신청서, 모회사 재무제표, 투자 유치 뉴스, 산업 데이터 등 다중 출처에서 짜맞춤
- **신뢰도 표기**: 모든 데이터 포인트에 🟢높음 / 🟡중간 / 🔴낮음 신뢰도 표기
- **다중 방법 밸류에이션 교차**: 라운드 밸류에이션법 + 유사기업 비교법 + DCF + 최종 국면 역산법
- **엑싯 경로 분석**: IPO/피인수/구주 거래 전 경로 평가

### 7. `/news-pulse` — 주가 이상 변동 뉴스 원인 규명

"급등/급락 시 무슨 일이 있었는지 빠르게 파악"하기 위해 설계된 정보 대응 Skill. **심층 리서치가 아니라 10-15분의 빠른 원인 규명이다** — 보유 종목의 이상 변동 시 소문에 휘둘리거나 맹목적 손절에 빠지는 것을 막는다.

**핵심 차별화**:
- **4차원 병렬 정찰**: 기업 이벤트 / 규제 정책 / 산업 경쟁사 / 시장 심리 (셀사이드+인플루언서+자금 흐름)
- **나열보다 원인 규명**: 모든 뉴스를 나열하는 게 아니라 "어떤 이벤트가 이 변동에 걸맞은가"를 판단
- **성격 판단 강제**: 가치 이벤트 / 심리 변동 / **진짜 원인 불명** / 혼합 — 그중 "진짜 원인 불명"이 가장 가치 있는 출력 (내부자 선행매매 가능성)
- **명확한 행동 제안**: 심층 리서치 트리거 여부, 논지 재심사 필요 여부, 관찰만 할지 등

**다른 Skill과의 구분**:
| 상황 | 무엇을 쓰나 |
|------|------|
| 완전한 투자 리서치 (시간 단위) | `/investment-team` 또는 `/investment-research` |
| 실적 정독 | `/earnings-review` |
| 장기 논지 추적 | `/thesis-tracker` |
| **주가 이상 변동 10분 원인 규명** | **`/news-pulse`** |

**출력 예시 발췌** (텐센트 4/17-5/01 실측, 14일간 -10.47%):

> #### 한 문장 원인 규명
> 이번 -10.47% 하락의 약 70-80%는 수급+심리 요인 (자사주 매입 침묵기 + 남향 자금 축소 + 섹터 베타 + AI 내러티브 상실), 20-30%는 AI 투자 2배 증가의 이연 소화가 부담 — **펀더멘털 악재 없음**, 셀사이드는 매수 컨센서스 유지. 성격상 "유동성+심리형 조정"이지 가치 이벤트가 아니다.
>
> #### 원인 규명 표
>
> | 후보 설명 | 추정 기여 | 신뢰도 |
> |---------|--------|--------|
> | 자사주 매입 침묵기 (구조적, 5/13 실적 전) | -3% ~ -4% | 상 |
> | 남향 자금의 텐센트 순매도 전환 | -2% ~ -3% | 상 |
> | AI 내러티브를 경쟁작에 빼앗김 (DeepSeek V4/Qwen3.6 등) | -1% ~ -2% | 중 |
> | 섹터/거시 베타 (유가+지정학+Fed 매파) | -2% ~ -3% | 상 |
> | 1분기 실적 전 리스크 회피 | -1% ~ -2% | 중 |
> | 펀더멘털 악화 | **0%** | 매우 높음 (배제) |
>
> #### 성격 판단: ✅ 혼합형
> 70% 수급/심리 + 20% AI 장기 내러티브 우려 + 10% 1분기 실적 전 불확실성
>
> **핵심 반증**: 돤융핑이 4/8 텐센트 풋 매도 (강세 관점); 셀사이드 24개사 컨센서스 Strong Buy; 넷이즈는 4/30 역행 상승 2% (게임 산업 문제 배제); 텐센트가 항셍테크를 7%p 언더퍼폼 (항셍테크는 월간 오히려 +4%).

호출 방식:

```
/news-pulse 텐센트
/news-pulse 핀둬둬 일주일간 -12%
/news-pulse 미호요
```

## 설계 이념

### 4대 거장 방법론의 융합

```
              ┌──────────────────┐
              │    돤융핑         │
              │  "옳은 사업"      │
              │  비즈니스 모델 본질│
              └────────┬─────────┘
                       │
    ┌──────────────────┼──────────────────┐
    │                  │                  │
    ▼                  ▼                  ▼
┌────────┐     ┌──────────┐      ┌────────┐
│  버핏   │     │   멍거    │      │  리루   │
│  해자   │     │ 역발상    │      │ 문명추세│
│ 안전마진│     │ 리스크목록│      │ 패러다임│
│ 경영진  │     │ 편향점검  │      │ 산업가치│
└────────┘     └──────────┘      └────────┘
```

네 거장은 단순한 분업이 아니라 **서로 도전하도록** 설계되었다:
- 돤융핑이 "좋은 사업"이라 하면 멍거는 "어떻게 죽는가"를 묻는다
- 버핏이 "충분히 싸다"고 하면 리루는 "10년 후에도 있을까"를 묻는다
- 당신이 받는 것은 네 보고서의 짜깁기가 아니라 네 가지 사고방식의 충돌이다

### 재무 엄밀성 도구 (`tools/financial_rigor.py`)

| 기능 | 명령 | 해결하는 문제 |
|------|------|-----------|
| **시총 검산** | `verify-market-cap` | 주가×총주식수 정밀 계산, 단위 오류 탐지 |
| **밸류에이션 검산** | `verify-valuation` | PER/PBR/ROE/FCF Yield 정밀 십진수 계산 |
| **다중 출처 교차 검증** | `cross-validate` | N개 출처의 같은 데이터 자동 대조, 허용 오차 초과 시 경보 |
| **3개 시나리오 밸류에이션** | `three-scenario` | 낙관/중립/비관 목표가 정밀 계산 |
| **벤포드 법칙 탐지** | `benford` | 재무 데이터 첫 자리 숫자 분포 이상 탐지 |
| **정밀 계산기** | `calc` | 임의의 재무 수식 정밀 계산, LLM 암산 대체 |

**설계 원칙**: 모든 계산은 Python `decimal.Decimal`(정밀 십진수)을 사용하며 `float`(부동소수점 근사)를 쓰지 않는다. `0.1 + 0.2 = 0.3`은 금융 상황에서 실패가 허용되지 않는다.
