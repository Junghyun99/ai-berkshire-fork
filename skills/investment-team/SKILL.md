# 투자 리서치 팀: 4개 역할 병렬 분석 프레임워크

$ARGUMENTS 에 대해 팀 기반 투자 리서치 분석을 수행한다. Team 도구로 진짜 멀티 Agent 병렬 리서치 팀을 만든다.

## 실행 절차

### 1단계: 팀 프레임워크 제시

사용자에게 아래 팀 구조를 보여주고, 확인 후 시작한다:

| 역할 | 책임 | 분석 프레임워크 |
|------|------|----------|
| **team-lead** (본인) | 총괄 조율, 종합 판단, 최종 보고서 출력 | 4대 거장 종합 프레임워크 |
| **business-analyst** | 비즈니스 모델 & 해자 분석 | 돤융핑 관점 |
| **financial-analyst** | 재무제표 & 밸류에이션 분석 | 버핏 관점 |
| **industry-researcher** | 산업 구도 & 경쟁 상황 | 멍거 관점 |
| **risk-assessor** | 리스크 평가 & 경영진 판단 | 리루 관점 |

### 1.5단계: AI 리서치 편향 평가

팀 생성 전, 해당 회사의 "AI 리서치 가능성" 평가를 사용자에게 먼저 보여준다:

**정보 풍부도 등급** (리서치 전략을 결정):
| 등급 | 특징 | 리서치 전략 조정 |
|------|------|------------|
| A등급 (정보 풍부) | 상장 오래됨, 증권사 커버리지 넓음 | 팀의 초점을 **반대편 검증**과 **비컨센서스 관점**에 둔다. 시장과 일치하는 "맞는 말이지만 쓸모없는 소리" 출력을 피한다 |
| B등급 (정보 보통) | 상장 얼마 안 됨, 커버리지 제한적 | 각 Agent의 추정 데이터에 신뢰도를 표기하고, team-lead가 취합 시 "데이터 충분도"를 표기 |
| C등급 (정보 희소) | 비주류/신규 상장/신흥 시장 | 팀을 "제1원리 모드"로 전환: 보고서 완결성을 추구하지 말고 사업 본질의 핵심 질문 몇 가지에 집중 |

**핵심 유의점**: 자료가 많다 ≠ 확실성이 높다, 자료가 적다 ≠ 확실성이 낮다. AI가 출력할 수 있는 신뢰도 ≠ 투자의 진짜 확실성. 확실성은 비즈니스 모델 자체에서 나오지, 자료의 양에서 나오지 않는다.

등급 결과를 각 Agent에게 알려 리서치 방식에 반영시킨다.

### 2단계: 팀 생성

TeamCreate로 팀을 만든다:
- team_name: `{회사명}-research` (영문 소문자, 예: `meituan-research`)
- agent_type: `team-lead`

### 3단계: 4개 태스크 생성

TaskCreate로 아래 4개 태스크를 만든다 (각각 subject, description, activeForm 포함):

#### 태스크 1: 비즈니스 모델 분석
- subject: `{회사명}의 비즈니스 모델, 해자, 사용자 가치 분석`
- description 포함 내용:
  1. 비즈니스 모델 본질: 핵심 사업 정의, 수익 구조 분해
  2. 플랫폼/제품 플라이휠 효과가 어떻게 돌아가는가
  3. 해자 분석: 브랜드/전환비용/네트워크 효과/규모의 경제/기술 장벽을 하나씩 검증
  4. 사용자/고객 가치: 각 이해관계자에게 어떤 고유 가치를 창출하는가
  5. 사업 매트릭스와 시너지 효과
  6. 돤융핑의 "좋은 사업" 기준 평가: 차별화, 가격결정력, 지속 가능한 경쟁 우위
  7. 최신 실적, 산업 보고서 등 공개 정보 검색 필수

#### 태스크 2: 재무 및 밸류에이션 분석
- subject: `{회사명}의 재무 데이터, 수익성, 밸류에이션 분석`
- description 포함 내용:
  1. 최근 3-5년 매출, 순이익, 영업이익 추세
  2. 수익성 지표: ROE, ROA, 매출총이익률, 영업이익률
  3. 현금흐름 분석: 영업현금흐름, 잉여현금흐름, 자본지출
  4. 재무상태표 건강도: 현금 보유, 부채비율, 유동성
  5. 밸류에이션 분석: PER/PSR/PBR/EV 등, 과거 및 동종업계 대비
  6. 안전마진 평가: 내재가치 vs 현재 주가
  7. **재무 엄밀성 검증 (반드시 Bash로 도구 호출, 암산 금지)**:
     - 시가총액 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap --price {가격} --shares {주식수} --reported {보고된 시총} --currency {통화}`
     - 밸류에이션 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation --price {가격} --eps {EPS} --bvps {주당순자산}`
     - 핵심 데이터 교차 검증: `python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate --field {필드} --values '{JSON}' --unit {단위}`
     - 3개 시나리오 밸류에이션: `python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario --price {가격} --eps {EPS} --shares {주식수(억)} --growth {낙관} {중립} {비관} --pe {낙관PE} {중립PE} {비관PE}`
     - 도구 출력 결과를 검증 기록으로 보고서에 그대로 삽입

#### 태스크 3: 산업 및 경쟁 분석
- subject: `{산업} 산업 구도와 {회사명}의 경쟁 상황 분석`
- description 포함 내용:
  1. 산업 규모와 성장: 시장 규모, 성장률, 침투율
  2. 경쟁 구도: 주요 경쟁사 시장 점유율, 경쟁 전략 비교
  3. 핵심 경쟁자 위협 평가: 주요 경쟁사를 하나씩 분석
  4. 세부 분야별 구도
  5. 산업 트렌드: 기술 변화, 정책 영향, 신규 진입자
  6. 밸류체인 분석: 상·중·하류 가치 배분
  7. 최신 산업 데이터와 경쟁 동향 검색 필수

#### 태스크 4: 리스크 및 경영진 평가
- subject: `{회사명}의 투자 리스크와 경영진 품질 평가`
- description 포함 내용:
  1. 경영진 평가: CEO의 능력 범위, 정직성, 전략적 안목, 자본 배분 능력, 과거 의사결정의 질
  2. 규제 리스크: 현재 및 잠재적 규제 영향
  3. 경쟁 리스크: 각 경쟁사의 위협 수준 평가
  4. 사업 리스크: 신사업 적자, 확장 불확실성
  5. 거시 리스크: 경기 사이클, 산업 사이클 영향
  6. 지배구조: 지분 구조, 특수관계자 거래, 주주환원 정책
  7. 장기 확실성: 10년 후 이 회사는 어떤 모습일까? 무엇이 비즈니스 모델을 무너뜨릴 수 있는가?
  8. 최신 규제 동향, 경영진 발언 등 검색 필수

### 4단계: 4개 Agent 병렬 실행

Task 도구로 4개 Agent를 동시에 실행한다 (**반드시 같은 메시지에서 병렬 호출**):

각 Agent의 설정:
- `subagent_type`: `general-purpose`
- `run_in_background`: `true`
- `team_name`: 해당 팀 이름
- `name`: 해당 역할명 (business-analyst / financial-analyst / industry-researcher / risk-assessor)

각 Agent의 prompt 템플릿:

```
당신은 {회사명} 투자 리서치 팀의 "{역할명}"으로, {거장 이름}의 투자 관점에서 {회사명}을 분석한다.

태스크 #{태스크 번호}를 완료하라: {태스크 subject}

구체적 요구사항:
{태스크 description의 내용}

**리서치 방법**:
- WebSearch로 최신 공개 정보(실적, 산업 보고서, 뉴스)를 검색
- **재무 데이터는 반드시 두 개의 독립 출처에서** 가져올 것. `skills/financial-data/SKILL.md` 규범에 따라 실행 (미국 주식: macrotrends+stockanalysis; 홍콩 주식: aastocks+macrotrends; 중국 A주: East Money+cninfo; 한국 주식: DART+네이버 금융). 두 출처 간 오차 >1%는 표기
- 데이터 정확성 확보, 핵심 데이터에 출처 표기
- 분석은 깊이 있게, 겉핥기 금지

**출력 요구사항**:
- 보고서는 상세하게, 핵심 데이터는 Markdown 표로 제시
- 각 분석 차원마다 명확한 결론과 점수
- 보고서 말미에 해당 차원의 종합 결론

**완료 후**:
1. TaskUpdate로 태스크 #{태스크 번호}를 completed로 표시
2. SendMessage로 전체 분석 보고서를 team-lead에게 전송 (type: "message", recipient: "team-lead")
```

### 5단계: 보고서 수신 및 진행 추적

- 사용자에게 진행 표를 실시간으로 보여준다 (어느 Agent가 완료됐고 어느 Agent가 아직 리서치 중인지)
- 보고서를 하나 받을 때마다 진행 상황을 갱신하고 그 보고서의 핵심 요점(3-5개)을 보여준다
- 4개 보고서가 모두 도착할 때까지 대기

### 6단계: 팀원 종료

보고서를 전부 받으면 4개 Agent에게 shutdown_request를 보낸다 (SendMessage 사용, type: "shutdown_request").

### 7단계: 최종 보고서 종합

4개 분석 보고서를 종합해 아래 구조의 최종 보고서를 출력한다:

---

#### 1. 한 문장 결론
> 투자할 가치가 있는지와 핵심 논리를 한 단락(50-100자)으로 요약

#### 2. 4차원 점수 총괄표
| 차원 | 프레임워크 | 점수(별 1-5) | 핵심 판단 |
|------|------|------------|----------|

종합 점수: X / 5

#### 3. 핵심 데이터 한눈에 보기
핵심 재무·경영 지표 표 (최근 2년 비교)

#### 4. 차원별 분석 요약
차원마다 가장 중요한 발견 3-5개 발췌

#### 5. 투자 논점 (Bull vs Bear)
- 🟢 강세 논리 (5-7개)
- 🔴 약세 논리 (5-7개)

#### 6. 버핏의 매수 전 Checklist
| # | 점검 항목 | 통과? | 설명 |
핵심 점검 항목 10개를 하나씩 평가

#### 7. 최종 투자 제안
- 정성 판단 표 (사업 품질/경영진/밸류에이션/타이밍)
- 유형별 조치 제안 표 (공격형/균형형/보수형 → 제안+가격 구간)
- 핵심 촉매 (비중 확대 신호/비중 축소 신호 각 3-5개)

#### 8. 요약 단락
100-200자의 최종 요약

### 8단계: 보고서 저장

완성된 최종 보고서를 `~/{회사명}-투자연구보고서_{날짜}.md`에 저장한다 (날짜 형식 YYYYMMDD).

```bash

  --report <보고서 파일 경로>

# Step 2 — 목록의 각 항목을 신뢰할 수 있는 출처에서 재확인 (skills/financial-data/SKILL.md 참조)

# Step 3 — 출고/반려 판정 출력

  --report <보고서 파일명>
```

**[출고]** 전부 통과 → 보고서 발행 가능; **[반려]** 미통과 항목 존재 → 수정 후 재심사.

### 10단계: 팀 정리

TeamDelete로 팀 리소스를 정리한다.

## 중요 주의사항

1. **4개 Agent는 반드시 병렬 실행** — 같은 메시지에서 Task 도구를 4번 호출
2. **Agent는 SendMessage로 보고** — 파일 협업이 아니라 메시지 통신이다
3. **데이터 정확성** — Agent가 WebSearch로 최신 데이터를 검색하고 핵심 데이터를 교차 검증하도록 요구
4. **결론은 명확하게** — 매수/관망/회피 제안과 구체적인 가격 구간 제시를 회피하지 않는다
5. **모든 분석에 데이터 근거** — 데이터 출처 첨부
6. **인내심 있게 대기** — 4개 Agent의 리서치에는 몇 분이 걸린다. 사용자에게 진행 상황을 실시간 갱신
7. **반편향 의식** — team-lead는 취합 시 반드시 평가한다: 각 Agent의 분석이 자료의 풍부함에 좌우되지는 않았는가? 시장 컨센서스에 과도하게 수렴하지는 않았는가? 최종 보고서에는 "정보 풍부도 등급"과 "AI 리서치 한계 고지"를 포함해야 한다
8. **정보 희소 시의 정직 원칙** — 추측으로 프레임워크를 채워 확실성을 가장하느니, 보고서에 여백을 남기고 "데이터 부족"을 표기하는 편이 낫다
