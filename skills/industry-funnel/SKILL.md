# 산업 퍼널 스크리닝: 전체 시장에서 3개 종목까지 가치투자 정선 프로세스

$ARGUMENTS 산업/방향에 대해 퍼널식 가치투자 스크리닝을 실행하여, 전체 시장 스캔에서 층층이 걸러 3개 최종 종목까지 정선한다.

## 적용 시나리오

하나의 산업이나 투자 방향(예: "AI 컴퓨팅", "신약", "로봇")을 말했을 때, 다음을 원하는 경우:
1. 중요한 종목을 하나도 놓치지 않는다 (한국/중국 A주/홍콩/미국 주식, 비상장 후보 포함)
2. 통일된 기준으로 "스토리 주식"과 품질 미달 기업을 걸러낸다
3. 정말로 심층 연구할 가치가 있는 선두 3개에 에너지를 집중한다
4. 각 층마다 명확한 잔류/탈락 기준이 있어 복기와 추적이 가능하다

`industry-research`와의 차이:
- `industry-research`는 밸류체인 구조와 전경에 중점, 단계별 절단면
- `industry-funnel`은 개별 종목 스크리닝 퍼널에 중점, 전체 시장에서 3개까지 정선

둘은 상호 보완적이다: 먼저 `industry-research`로 밸류체인 구도를 파악하고, 그다음 `industry-funnel`로 종목을 정선한다.

## 퍼널 구조 총람

```
1층: 전체 시장 스캔      30-60개   (활발도+상승률+시가총액 상위 30의 합집합)
        ↓ 가치투자 5대 정량 지표
2층: 1차 스크리닝          ≤ 10개   (5개 전부 합격 + 해자 ★★★ 이상)
        ↓ 정밀 분석
3층: 정밀 분석             ≤ 10개  (기업당 300-500자 구조화 분석)
        ↓ 최종 선정
4층: 4대 거장 심층 분석      3개     (기업당 800-1200자, 버핏·멍거·돤융핑·리루 관점)
        ↓
출력: 투자 제안 + 매매 신호 + 비중 제안
```

각 층에서 "걸러진 종목"에는 반드시 탈락 이유를 남긴다. 블랙박스 금지.

## 1단계: 전체 시장 스캔 입구

### 1.1 활성 종목 정의 (3개 유형의 합집합)

**A류 - 거래 활발도**:
- 최근 30일 일평균 거래대금이 산업 상위권 (시장별로 각각 상위 30)

**B류 - 상승률 순위**:
- 최근 30일 상승률 상위 20
- 최근 90일 상승률 상위 20
- 둘의 합집합

**C류 - 시가총액 앵커**:
- 산업 내 시가총액 상위 30 (등락 무관)

최종 스캔 풀 = A ∪ B ∪ C, 예상 30-60개.

### 1.2 반드시 검색할 시장

| 시장 | 출처 제안 |
|------|---------|
| 한국 (KOSPI/KOSDAQ) | 네이버 금융, KRX 산업 분류 |
| 중국 A주 (상하이/선전) | East Money 산업 섹터 |
| 홍콩 주식 | HKEX 산업 분류 |
| 미국 주식 | NASDAQ/NYSE 산업 ETF 보유 종목, Yahoo Finance |
| 국제 시장 | 일본·대만·유럽의 관련 기업을 놓치지 말 것 (특히 반도체, 전자) |
| 비상장 기업 | "미래 IPO 후보" 소절을 별도로 두고 최신 밸류에이션과 잠재 IPO 시기 명기 |

### 1.3 출력 형식

| 회사명 | 코드 | 시장 | 시가총액 | 주력 사업 한 문장 | 해당 산업 비중 | 선정 유형(A/B/C) |
|-------|------|-----|------|----------|-----------|----------------|

**핵심 자가점검**:
- 산업 비중 < 30%인 "걸치기 주식"은 신중하게, "순수 종목 아님" 표기
- 영어 자료가 적다고 한국/아시아 시장을 놓치지 말 것
- AI의 선두 기업 선호 때문에 소형주를 놓치지 말 것

## 2단계: 가치투자 5대 정량 지표 1차 스크리닝 → ≤ 10개

1단계의 30-60개 기업에 5개 정량 지표를 하나씩 적용한다.

### 2.1 5대 정량 지표

| # | 지표 | 통과 기준 | 완화 조건 | 데이터 출처 |
|---|------|---------|---------|---------|
| 1 | PER 밸류에이션 | 적정 (과거 구간, 동종업계 대비) | 고성장은 PEG < 1.5까지 완화 가능 | 재무제표+데이터 플랫폼 |
| 2 | ROE | > 15% 또는 최근 3년 추세 개선 | 중자산 산업은 완화 가능 | 재무제표 |
| 3 | 영업현금흐름 | 플러스이고 순이익의 > 70% | — | 재무제표 |
| 4 | 부채비율 | < 60% | 유틸리티/전력은 70%까지 완화 | 재무제표 |
| 5 | 해자 속평 | ★★★ 이상 | — | 정성 판단 |

**해자 5가지 유형**:
- 브랜드/가격결정력
- 전환 비용/사용자 점착성
- 네트워크 효과
- 규모의 경제
- 기술/라이선스/자원 장벽

### 2.2 출력 형식

| 회사 | PER | ROE | 현금흐름/순이익 | 부채비율 | 해자 | 종합 | 잔류/탈락 | 탈락 이유 |
|------|----|----|-----------|-------|-------|------|------|---------|

**잔류 규칙**:
- 5개 전부 합격 → 바로 잔류
- 4개 합격 + 1개 근접 → 잔류하되 노란색 표시
- 4개 미만 → 탈락, 이유 명기

**목표**: ≤ 10개 잔류. 너무 많이 남으면 (> 12) 해자 기준을 ★★★★로 올려 다시 거른다.

## 3단계: 정밀 분석 (≤ 10개, 기업당 300-500자)

1차 스크리닝을 통과한 기업을 하나씩 구조화 분석한다.

### 3.1 기업별 분석 템플릿

```
## {회사명} ({코드})

**한 문장 비즈니스 모델**:
(무엇을, 누구에게 팔고, 어떻게 돈을 받는가)

**재무 품질**:
- 매출 성장률 / 이익 성장률 / 매출총이익률 / ROE / 현금흐름
- 핵심 변화 (최근 1-2년의 가장 중요한 재무 변곡점)

**해자 깊이**:
- 주요 해자 유형 + 구체적 증거
- 5년 후에도 해자가 남아 있을지 간단 판단

**주요 리스크 (상위 3)**:
1.
2.
3.

**밸류에이션 속평**:
- 현재 PER/PSR/EV/EBITDA + 과거 구간 내 위치
- 동종업계 비교
- 한 문장 결론: 비쌈 / 적정 / 저렴

**최종 3개 진입?**: 예 / 아니오 (이유)
```

### 3.2 최종 3개 선정 기준

점수 순위로 상위 3개를 뽑는 게 아니라 "포트폴리오 상호보완성"으로 뽑는다:
- 최소 1개는 "고확실성 저탄력" (버핏형)
- 최소 1개는 "중간 확실성 중간 탄력" (성장형)
- 선택적으로 1개 "고탄력 고위험" (옵션형)

어떤 세부 분야에서 충분히 좋은 3개를 못 찾으면 "최종 2개 + 관찰 1개"로 쓰는 편이 낫다. 머릿수 채우기 금지.

## 4단계: 4대 거장 심층 분석 (3개, 기업당 800-1200자)

최종 3개에 4대 거장 관점의 심층 분석을 실행한다.

### 4.1 돤융핑 관점: 사업 본질

- 이 회사가 무슨 사업을 하는지 한 문장으로 정의
- 좋은 사업인가? 왜?
- 이 회사의 "본분"은 무엇인가? 경영진이 벗어나지는 않았는가?
- 비즈니스 모델의 "지속성"은 어디에 있는가?

### 4.2 버핏 관점: 해자 깊이

- 다섯 가지 해자로 채점 (★1-5), 구체적 증거 나열
- 10년 후에도 해자가 남아 있을까?
- 지금 매수할 때의 "안전마진"은 어디에 있는가?

| 해자 | 강도 | 구체적 증거 |
|-------|------|---------|
| 브랜드/가격결정력 | | |
| 전환 비용 | | |
| 네트워크 효과 | | |
| 규모의 경제 | | |
| 기술/라이선스 장벽 | | |

### 4.3 멍거 관점: 리스크와 실패 패턴

- 이 회사는 어떻게 실패할 가능성이 가장 큰가? (상위 3개 실패 경로)
- 최악의 시나리오에서 얼마짜리인가? (초간단 밸류에이션)
- 똑똑한 사람들은 왜 안 사는가? (역방향 논증)
- 도덕/컴플라이언스/경영진 리스크가 있는가?

### 4.4 리루 관점: 문명 수준 트렌드 포지셔닝

- 이 회사가 속한 분야는 "문명 수준의 패러다임 전환"인가 "일시적 열풍"인가?
- 역사상 가장 가까운 기술 혁명 유추는?
- 10-20년 후 이 회사의 최종 국면은?
- 승자독식 구도인가?

### 4.5 종합 추천도

```
추천도: ★★★★☆
비중 유형: 핵심 / 위성 / 옵션 / 관찰
권장 매수 구간: 현재가 / N% 조정 시 / 인내심 있게 대기
권장 보유 비율: 해당 테마 비중의 X%
핵심 모니터링 지표: (이 회사의 논리가 뒤집히는 신호는 무엇인가)
```

## 5단계: 종합 출력

보고서 말미에 통합한다:

### 5.1 최종 3개 포트폴리오 표

| 회사 | 유형 | 추천도 | 권장 비중 | 핵심 논리 | 핵심 리스크 |
|------|------|-------|---------|---------|---------|
| A | 핵심 | ★★★★★ | 50-60% | | |
| B | 위성 | ★★★★☆ | 25-35% | | |
| C | 옵션 | ★★★☆☆ | 5-15% | | |

### 5.2 산업 수준 ETF 대체

종목 선정이 싫으면 관련 ETF 1-3개를 나열한다 (한국/홍콩/미국).

### 5.3 산업 전반의 위치 판단

- 산업 PER/PBR 과거 분위
- 자금 흐름 (외국인 자금, ETF 설정·환매, 셀사이드 커버리지 밀도)
- 전체적으로 "초기/확장/성숙/쇠퇴" 중 어느 단계인가

### 5.4 정보 충분도 자가평가 (필수)

| 차원 | 등급 | 설명 |
|-----|------|-----|
| 기업 재무 데이터 완전성 | A/B/C | |
| 밸류에이션 데이터 시의성 | A/B/C | |
| 산업 구도 판단 | A/B/C | |
| 경영진 정보 | A/B/C | |

A = 데이터 충분하고 신뢰 가능; B = 일부 누락되나 주 결론에 영향 없음; C = 누락 많음, 결론에 신중해야 함.

### 5.5 업데이트 대기 데이터 포인트

명확히 나열한다: 어떤 데이터가 추정값인지, 어떤 데이터가 후속 확인이 필요한지, 어느 분기 실적을 중점 추적해야 하는지.

### 5.6 자료 출처 목록

각 데이터/결론의 출처 링크를 분류해 나열한다 (재무제표, 리포트, 뉴스, 산업 보고서).

## AI 리서치 편향 자각 (중요)

퍼널 스크리닝 과정에서 AI가 빠지기 쉬운 함정:

| 편향 | 양상 | 대응 |
|------|-----|------|
| 선두 기업 선호 | 대형주는 자료가 많고 분석 분량이 길어 "더 좋아" 보임 | 보고서 분량이 아니라 정량 지표와 해자 점수로 순위 |
| 영어 선호 | 미국 주식 자료가 풍부해 한국/홍콩 주식이 저평가되기 쉬움 | 반드시 여러 언어로 검색, 아시아 기업을 놓치지 말 것 |
| 스토리 선호 | 높은 상승률 + 미디어 열기 = 더 좋아 보이는 "AI 테마주" | "AI 매출 비중" vs "AI 스토리 비중" 구분, 실제 사업을 볼 것 |
| 현재 선호 | 지금 재무가 좋은 기업이 선정되기 쉬워 전환기 다크호스를 놓칠 수 있음 | 2층 스크리닝에서 "추세 개선"을 완화 조건으로 허용 |
| 상장사 선호 | 상장사만 보면 분야 최고의 플레이어를 놓칠 수 있음 | "미래 IPO 후보"를 반드시 나열, 밸류에이션과 시간 창 표기 |

## 출력 요구사항

1. **보고서 위치**: `reports/{산업명}-funnel-{YYYYMMDD}.md` (산업 보고서는 reports/ 루트에)
2. **언어**: 한국어
3. **스타일**: 직접적이고 예리하게, 군말 없이
4. **데이터**: 모든 데이터에 출처 표기; 추정값은 "추정" 표기
5. **입장을 미리 정하지 않는다**: 데이터 제시 → 논리 전개 → 결론 도출
6. **양면 제시**: 모든 핵심 판단에 반대 논거 첨부
7. **층마다 탈락 기록**: 탈락한 기업도 이름+이유를 남긴다

## 데이터 표본 검사 (출고 프로세스)

보고서 저장 후 데이터 표본 검사를 실행하고, 통과해야 발행할 수 있다:

```bash
# Step 1 — 표본 검사 목록 추출 (15% 무작위 샘플링)
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>

# Step 2 — 목록의 각 항목을 신뢰할 수 있는 출처에서 재확인 (skills/financial-data/SKILL.md 참조)

# Step 3 — 출고/반려 판정 출력
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<채워진 JSON>' \
  --report <보고서 파일명>
```

**[출고]** 전부 통과 → 보고서 발행 가능; **[반려]** 미통과 항목 존재 → 수정 후 재심사.
