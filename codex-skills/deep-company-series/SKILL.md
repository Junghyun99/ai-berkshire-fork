## 3. 문체 규범

### 어조

- **직접적이고 예리하게, 군말 없이** — 첫 문장부터 숫자나 반직관적 결론을 제시
- **가치투자 프레임워크** — 버핏/멍거/돤융핑/리루 관점을 녹여 넣되 명언을 나열하지 않는다
- **입장을 미리 정하지 않는다** — 데이터 먼저, 논리 전개, 마지막에 결론
- **양면 제시** — 모든 핵심 판단에 "그러나 다른 한편으로..."라는 반론을 붙인다
- **모바일 가독성** — 첫 18-20자만으로도 내용이 서야 한다 (휴대폰 미리보기)

### 금지 표현

| 금지 | 이유 | 대체 |
|------|------|------|
| 명백히 / 필연적으로 / 반드시 | 주관적 단정 | 데이터에 따르면 / 증거가 보여주듯 |
| 나는 ~라고 생각한다 / 느낀다 | 주관적 어조 | 삭제하거나 "본 프레임워크 기준"으로 |
| 교과서급 / 신의 한 수 | 어그로성 찬사 | 구체적 사실 서술 |
| 심각한 불일치 / 심각한 저평가 | 강한 주관어 | 구체적 할인율 %를 제시 |
| 완벽하다 / 흠잡을 데 없다 | 일방적 판단 | 반대편 관찰을 덧붙임 |

### 제목 스타일

- **반전 숫자**나 **반공감대 결론**을 훅으로 쓴다 ("15년간 7번 도전 전부 실패", "연봉 4,292만이 이익의 0.0017%")
- 부제는 중립적으로 내용을 요약 ("— `<본질 판단>`")
- **어그로성 비유 금지**: "리틀 버핏", "한국판 X", "갓X" 일체 금지
- 전문 독자에게 익숙한 용어 사용 ("버크셔"이지 "버핏"이 아님, 인명보다 회사명 우선)

## 4. 엄격한 팩트체크 체크리스트 (핵심 역량)

### 쓰기 전부터 경계해야 할 "가짜 정밀함" 함정

1. **확률 가중 기댓값**: `30% × A + 50% × B + 20% × C = 기대 +X%` 류의 계산은 거의 전부 쓰레기다 — 확률 배분이 순수 주관이라 독자에게 잘못된 정밀감을 준다. **시나리오 + 트리거 조건 + 방향만 제시하고, 가중 기댓값은 계산하지 않는다.**
2. **제3자 추정 MAU/점유율**: 데이터 업체별 기준 차이가 극심하다 (같은 시점에 2-3배 차이 가능). **가장 신뢰할 만한 두 개만 앵커로 비교하고 나머지는 정성 서술.**
3. **과거 성장률 선형 외삽**: `2025년 +33% × 5년 복리 → 2030년 X`는 금융 문맹식 예측이다. **시나리오 가정 + 상/하단 구간 + 약속이 아님을 명시.**
4. **비공개 지분율**: 바이트댄스류 비상장사 지분은 **공개 공시된 적이 없다**. **구간으로 제시하고 "확인 불가" 표기.**
5. **강한 인과 귀속**: 경쟁사 실패 = X 때문. 복수의 원인을 모두 나열하고 **본문에서 단일 귀속을 하지 않는다.**

### 수정 시 반드시 돌리는 7가지 점검

```
□ 1. 편간 숫자 일관성: 시가총액, Non-IFRS 순이익, 핵심 지분율이 시리즈 전체에서 일치
□ 2. 기준 표기: Non-IFRS / GAAP / Non-IFRS-SBC / FCF 중 무엇을 썼는지 전체적으로 명확
□ 3. 중복 계산 스캔: 연결 자회사가 "투자 포트폴리오"에 중복 편입되지 않고, SOTP 이중계산 없음
□ 4. 횡단 비교 공정성: "주력사업 PER(현금+포트폴리오 차감)" vs "경쟁사 PER(미차감)" 식 비교 금지
□ 5. 확률 가중치 전부 삭제: 위 항목 참조
□ 6. 단정 표현 전부 완화: grep "명백|필연|심각|교과서|완벽"
□ 7. 제3자 데이터 출처 표기: 재무제표 외 모든 데이터 뒤에 "(출처: X)"
```

### 모델이 자주 틀리는 부분

쓰기 전에 **알려진 하드 에러 리스크를 먼저 나열한다**:
- 과거 수익 배수: 반드시 누적 투입 기준으로 계산 (예: Riot은 33배이지 58배가 아님)
- 지분율: 반드시 최신 공시/재무제표 기준 확인 (예: 텐센트의 메이투안 지분은 1.5%이지 6.4%가 아님)
- "현물 배당" 회계 처리: 간주 처분이익은 IFRIC 17에 따라 배당 선언일에 인식 (예: JD는 2021년, 메이투안은 2022년이나 금액 작음)
- 총주식수 반등: SBC가 연초에 집중 부여되면 주식수가 단기 상승할 수 있음

- 사용자의 수정 의견을 받아 수정

### 단계 3: 편간 일관성 스캔 (8편 전체 완성 후)

1. 같은 숫자(시가총액, 순이익, 지분율)가 편마다 일치하는가
2. 같은 용어(FBS, SBC, Non-IFRS)가 첫 등장 시 설명되었는가
3. 상호 참조: 02편의 "자세한 내용은 06편"이 실제로 대응하는가
4. 요점 정리 vs 본문의 숫자가 일치하는가

```bash

grep -r "/Users/\|<사용자 개인 식별 정보>" reports/ | head
```

## 6. 수정 의견 처리 절차

사용자가 수정 의견을 주면 아래 순서로 처리한다:

### 1. 먼저 사실 확인 (바로 고치지 않는다)

사용자가 "X 데이터가 틀렸다"라고 하면, 먼저 Bash/Read로 원본 데이터를 찾아 교차 검증한다:
- 이 프로젝트 내 같은 회사의 earnings/실적 보고서 확인
- 공식 공시 확인
- "사용자가 말한 데이터 vs 내가 찾은 데이터 vs 내가 기존에 쓴 데이터" 3자 비교 제시

### 2. 수정 등급 판단

| 등급 | 유형 | 처리 |
|------|------|------|
| 🔥 하드 에러 | 숫자 오류, 귀속 오류, 기준 오류 | 무조건 수정, 망설임 불필요 |
| ⚠️ 주관화 | 강한 주관어, 단정, 어그로성 비유 | 완화 또는 삭제 |
| 🔬 세밀도 | 출처 표기, 기준 세분화 | 우선순위 낮음, 가독성과 균형 |
| ❓ 신뢰 불가 | 제3자 추정치 편차 큼 | **고치기보다 삭제가 안전** (사용자 명시 지시) |

### 3. 수정 후 연쇄 점검

한 곳을 고치면 먼저 "이 숫자/개념을 인용하는 다른 곳이 어디인가"를 생각한다. 예:
- 시가총액 수정 → 시리즈 전체의 PER / 주력사업 PER / 할인율 / FCF Yield 연쇄 수정
- 지분율 수정 → TOP 10 순위 + 과거 지분 표 + 매도 리스트 수정
- 용어 기준 수정 → 최초 정의 + 이후 인용 + 요점 정리 수정

### 4. 푸시 후 즉시 보고

```
푸시 성공 (commit hash).
[N]건 수정 요약 [표 포함]:
- 무엇을 고쳤는지
- 연쇄로 무엇을 고쳤는지
- 아직 안 고친 것은 무엇인지

다음 지시 대기.
```

## 7. 이 skill이 하지 않는 것

- **독자를 대신해 투자 결정을 내리지 않는다** — 모든 편의 말미에 "투자 조언이 아닙니다"
- **주가를 예측하지 않는다** — "시나리오 + 트리거 조건"만 제시
- **"기대 연수익률" 가중값을 계산하지 않는다** — 주관적 확률 배분은 독자를 오도한다
- **"거물 X도 보유 중"이라고 쓰지 않는다** — 타인의 보유로 자기 판단을 뒷받침하는 것은 가치투자에 반한다
- **8편을 억지로 채우지 않는다** — 어떤 편에 독립적 내용이 부족하면 (예: 경영진이 특별할 게 없는 회사) 다른 편에 합치거나 편수를 줄인다

## 한 문장 요약

**"X 이해하기 시리즈"의 핵심 역량 ≠ 잘 쓰기, = 엄격하게 고치기** —
경제·금융 장문의 89%는 가짜 정밀 숫자, 주관적 가중 기댓값, 단정적 표현으로 죽는다. 이 skill의 존재 이유는 이 함정들을 전부 표시해서, 쓰기 전에 피하고 쓴 후에 청소하는 것이다.
