# 투자 리서치: 버핏-멍거-돤융핑-리루 4대 거장 종합 분석 프레임워크

$ARGUMENTS 에 대해 체계적인 투자 리서치 분석을 수행한다.

## 리서치 프레임워크

버핏, 멍거, 돤융핑, 리루 네 투자 거장의 방법론을 바탕으로 아래 7개 모듈을 순서대로 실행한다:

### 사전 단계: AI 리서치 편향 자각 (필수 실행)

리서치 시작 전에 먼저 해당 회사의 "AI 리서치 가능성"을 평가하고 잠재적 데이터 편향을 식별한다:

**정보 풍부도 등급**:
| 등급 | 특징 | AI 리서치 함정 | 대응 전략 |
|------|------|-----------|---------|
| A등급 (정보 풍부) | 상장 오래됨, 증권사 커버리지 많음, 미디어 보도 밀집 | 컨센서스가 너무 강해 AI 출력이 시장 가격에 수렴하며 alpha 제한적 | 반대편 검증에 집중: 똑똑한 사람들은 왜 안 사는가? 간과된 리스크는 무엇인가? |
| B등급 (정보 보통) | 상장 1-3년, 커버리지 제한적, 일부 데이터 추산 필요 | AI가 "합리적 추측"으로 공백을 채워 완전해 보이지만 실은 가짜 확실성 | 추산 데이터마다 신뢰도 표기, "근거 있는 추산"과 "허공 채우기" 구분 |
| C등급 (정보 희소) | 신규 상장/비주류/신흥 시장, 커버리지 거의 없음 | AI가 자료 부족으로 과도하게 보수화되어 "안 보임=나쁨"으로 오판 | 제1원리 질문 사용(아래 참조), 제한된 정보에서 사업 본질을 추출 |

**C등급 회사의 제1원리 리서치법**:
공개 자료가 부족할 때는 "완전해 보이는" 보고서를 짜맞추려 하지 말고 아래 근본 질문에 집중한다:
1. 고객은 누구인가? 왜 돈을 내는가? 대체 선택지는 없는가?
2. 재구매는 무엇으로 움직이는가? 습관인가, 락인인가, 지속적인 새 가치 창출인가?
3. 경쟁사가 100억을 들고 오면 이 사업을 복제할 수 있는가?
4. 경영진은 어떤 핵심 결정을 내렸는가? 그 결정들은 어떤 판단력과 가치관을 보여주는가?

**편향 자가점검 리스트** (리서치 내내 경계 유지):
- [ ] 내 "확실함"의 느낌은 사업 본질에서 오는가, 자료의 양에서 오는가?
- [ ] 이 회사의 자료량이 절반으로 줄면 내 결론이 바뀌는가?
- [ ] AI가 출력한 분석이 시장 컨센서스와 판박이는 아닌가? 그렇다면 내 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 사업 본질은 지극히 좋은" 가능성이 저평가되고 있지는 않은가?

정보 풍부도 등급 결과를 보고서 서두에 기재하고, 최종 결론에서 "AI 리서치 신뢰도"와 "실제 투자 확실성"의 차이를 명시한다.

### 1단계: 데이터 수집

> **데이터 소스 규범**: `.claude/skills/financial-data/SKILL.md` 참조. 모든 재무 데이터는 두 개의 독립 출처에서 가져와야 하며, 오차 >1%는 표기.
> - 미국 주식: macrotrends (주) + stockanalysis (부)
> - 홍콩 주식: aastocks (주) + macrotrends ADR (부)
> - 중국 A주: East Money (주) + cninfo (부)
> - 한국 주식: DART (주) + 네이버 금융 (부)

Task 도구로 백그라운드 Agent를 띄워 웹에서 아래 데이터를 수집한다:

1. 매출 구조: 최근 회계연도 및 최근 4개 분기의 부문별 매출, 성장률, 매출총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 매출총이익률, 영업이익률, 잉여현금흐름, 현금 보유
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 비교
4. 비즈니스 모델과 해자: 핵심 경쟁 우위의 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분율, 핵심 의사결정 기록
7. 산업 전망: TAM(총 도달 가능 시장), 성장 전망
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PER, PSR, PEG, EV/Revenue
10. 강세·약세 양측의 핵심 논점

#### 데이터 교차 검증 (필수 실행, 재무 엄밀성 도구 사용)

데이터 수집 완료 후, **반드시 `tools/financial_rigor.py`를 호출해 핵심 데이터를 프로그램으로 검증한다**. LLM 암산 오차를 원천 차단한다.

**반드시 검증할 데이터 포인트**:
- 총주식수 (거래소, Yahoo Finance, StockAnalysis 등 최소 2개 출처에서 확인)
- 현재 주가와 시가총액 (**주가×총주식수를 직접 계산해 보고된 시총과 대조, 단위 오류 방지**)
- 최근 회계연도 매출과 순이익 (회사 연차보고서 + 최소 1개의 제3자 출처에서 확인)
- 현금 보유와 순현금 (현금+단기투자-총부채, 기준 차이에 유의)
- 경영진 지분율 (경제적 지분과 의결권 구분, 차등의결권 구조 유의)

**강제 검증 단계 (Bash로 도구 호출)**:

Step 1 — 시가총액 검산 (정밀 십진수, 부동소수점 아님):
```bash
python3 tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총주식수} --reported {보고된 시총} --currency {통화}
```

Step 2 — 핵심 데이터 다중 출처 교차 검증:
```bash
python3 tools/financial_rigor.py cross-validate \
  --field {필드명} --values '{"출처1": 수치, "출처2": 수치}' --unit {단위}
```
매출, 순이익, 현금 보유에 대해 각각 실행.

Step 3 — 밸류에이션 지표 정밀 검산 (PER/PBR/ROE/FCF Yield 등):
```bash
python3 tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당순자산} --fcf-per-share {주당 FCF} --dividend {주당 배당}
```

**검증 규칙**:
1. 핵심 데이터 포인트마다 최소 2개의 독립 출처
2. 출처 간 차이 발견 시 회사 연차보고서/거래소 데이터를 우선 채택하고 차이 원인 명기
3. **계산이 필요한 모든 데이터는 도구로 검산해야 하며 LLM 암산 금지**
4. 도구 출력 결과를 보고서 부록 "핵심 데이터 교차 검증 기록"에 그대로 삽입
5. 도구가 ❌ 편차 과대를 보고하면 반드시 원인을 규명한 후에야 분석을 계속할 수 있다

**흔한 오류 방지**:
- 시가총액 단위: 홍콩달러 억 vs 위안 억 vs 달러 억 vs 원화 조, 0 하나를 빼먹거나 더 쓰기 쉽다
- FCF 기준: 출처마다 자본지출 정의가 다를 수 있다 (리스, 인수 포함 여부 등)
- 부채 기준: 운용리스 부채 포함 여부
- 지분율: 차등의결권 회사의 경제적 지분 ≠ 의결권

### 2단계: 사업 본질 분석 — 돤융핑의 "옳은 사업"

분석 요점:
- 이 사업의 본질을 한 문장으로 정의
- 매출 구조 분해 (차트)
- 5년 수익성 추세 (차트)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 점착성/고객 락인 강도
- 매출총이익률 수준과 동종업계 비교, 왜 높은지/낮은지 설명
- 영업 레버리지 분석
- **돤융핑식 되물음**: 이 사업의 좋은 점은 어디인가? 한 문장으로만 표현한다면 무엇인가?

### 3단계: 해자 평가 — 버핏의 "경제적 해자"

다섯 가지 해자를 하나씩 검증한다:

| 해자 유형 | 검증 방법 |
|-----------|---------|
| 브랜드/가격결정력 | 판매량 손실 없이 가격을 올릴 수 있는가? |
| 전환 비용 | 고객이 경쟁 제품으로 옮기는 비용이 얼마나 큰가? |
| 네트워크 효과 | 사용자가 많을수록 제품이 좋아지는가? |
| 규모의 경제 | 규모가 가져오는 비용 우위가 얼마나 큰가? |
| 기술/특허 장벽 | 기술이 몇 년 앞서 있는가? 복제 가능한가? |

해자 추세 분석: 지난 5년간 넓어졌는가 좁아졌는가? 향후 5년 전망.

**버핏식 되물음**: 10년 후에도 이 해자가 남아 있을까? 무엇이 이것을 무너뜨릴 수 있을까?

### 4단계: 역발상과 리스크 목록 — 멍거의 "거꾸로 생각하라"

- "이 회사가 실패할 수 있는 모든 경로"를 나열 (표: 경로/확률/영향 정도)
- 역사적 유추: 역사상 비슷한 위치에 있던 회사를 찾아 결말이 어땠는지
- 학제간 분석: 네트워크 효과 이론, 기술 수용 곡선, 경쟁 게임이론 등 모델로 교차 검증
- 편향 자가점검: 내러티브 편향, 앵커링 효과, 생존자 편향
- 약세론자의 핵심 논점 수집

**멍거식 되물음**: 나는 어디서 가장 틀리기 쉬운가? 똑똑한 사람들은 왜 이 회사를 안 사거나 공매도하는가?

### 5단계: 경영진 평가 — 돤융핑의 "옳은 사람" + 버핏의 "경영진 정직성"

- CEO/창업자의 핵심 의사결정 복기 (표: 시기/결정/결과/점수)
- 자본 배분 능력: R&D 수익률, 인수 성공률, 자사주 매입 타이밍
- 주주 이익 일치성: 경영진 지분, 보수 구조, 지분 매각 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특징

**돤융핑식 되물음**: CEO가 은퇴해도 이 회사는 경쟁력을 유지할 수 있는가?

### 6단계: 산업과 문명 트렌드 — 리루의 "문명 진화 프레임워크"

- 해당 산업이 "문명 수준의 패러다임 전환"에 있는지 판단
- 역사적 기술 혁명과의 유추 (증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 천장 분석
- 산업 밸류체인에서 회사의 위치
- 기술 노선 리스크
- 고객/공급사 집중도 분석

**리루식 되물음**: 20년 후에 돌아보면, 이 회사는 "이 시대의 스탠더드오일"인가 "반짝하고 사라진 3Com"인가?

### 7단계: 밸류에이션과 안전마진 — 버핏의 "내재가치" + 돤융핑의 "옳은 가격"

- 현재 시장 가격 (핵심 밸류에이션 지표 표) — **반드시 도구로 검산**
- 역방향 DCF: 현재 주가에는 어떤 성장 기대가 내포되어 있는가?
- 3개 시나리오 밸류에이션 — **반드시 도구로 정밀 계산, 암산 금지**:
```bash
python3 tools/financial_rigor.py three-scenario \
  --price {주가} --eps {EPS} --shares {총주식수(억)} \
  --growth {낙관 성장률} {중립 성장률} {비관 성장률} \
  --pe {낙관PE} {중립PE} {비관PE} --years 3 --currency {통화}
```
- 자사 과거 밸류에이션과 비교
- 동종업계 밸류에이션과 비교

**돤융핑식 되물음**: 내일부터 주식시장이 5년간 문을 닫는다면, 이 가격에 보유할 의향이 있는가?

## 출력 요구사항

1. 모든 분석에 데이터 근거와 출처 첨부
2. 핵심 데이터는 Markdown 표로 제시
3. 각 모듈 말미에 해당 거장의 "되물음" 포함
4. 최종적으로 완전한 보고서를 `reports/{회사명}/{회사명}-research-{YYYYMMDD}.md`에 저장 (CLAUDE.md 명명 규칙)
5. 결론은 명확하게. 매수/관망/회피 제안을 회피하지 않는다
6. 밸류에이션 부분은 구체적인 가격 구간을 제시해야 한다
7. **보고서 서두**에 "정보 풍부도 등급"(A/B/C)과 "AI 리서치 한계 고지"를 포함해야 한다
8. **보고서 말미**에 "AI 분석 신뢰도"와 "투자 확실성"을 구분해야 한다 — 전자는 자료량에 달려 있고 후자는 사업 본질에 달려 있다. 이 보고서의 어떤 결론이 충분한 데이터에 기반하고 어떤 결론이 제한된 정보의 추론에 기반하는지 독자에게 명확히 알린다
9. C등급(정보 희소) 회사라면 보고서 말미에 "1차 검증이 필요한 질문 목록"을 나열해야 한다 — 현장 조사, 제품 체험, 공급망 인터뷰 등으로 AI의 사각지대를 보완하도록 독자에게 권한다

## 데이터 표본 검사 (출고 프로세스)

보고서를 파일에 저장한 후 **반드시** 데이터 표본 검사를 실행하고, 통과해야 발행할 수 있다:

**Step 1 — 표본 검사 목록 추출 (15% 무작위 샘플링):**
```bash
python3 tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿을 출력하며, 각 항목에 `fetched_value`(채울 값)가 있다.

**Step 2 — 데이터 재확인:**
목록의 각 데이터 포인트를 `.claude/skills/financial-data/SKILL.md` 규범에 따라 신뢰할 수 있는 출처에서 가져온다
(미국: macrotrends+stockanalysis; 홍콩: aastocks+macrotrends; 중국 A주: East Money+cninfo; 한국: DART+네이버 금융).
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2`를 채운다.

**Step 3 — 판정 출력:**
```bash
python3 tools/report_audit.py verdict \
  --results '<채워진 JSON>' \
  --report <보고서 파일명>
```

- **[출고]**: 모든 표본 검사 포인트의 편차 ≤ 1% → 보고서 발행 가능
- **[반려]**: 어느 포인트든 편차 > 1% → 해당 데이터를 수정한 후 재검사, 출고될 때까지 반복
