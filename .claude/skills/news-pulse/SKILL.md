# 기업 뉴스 펄스: 주가 이상 변동 빠른 원인 규명 팀

$ARGUMENTS 에 대해 최근 뉴스 정찰과 이상 변동 원인 규명을 수행한다. **이것은 심층 투자 리서치가 아니라 정보 신속 대응이다** — 목표는 10분 안에 답하는 것: "이 회사에 최근 무슨 일이 있었나? 주가 이상 변동의 진짜 원인은? 투자 논지를 재심사해야 하나?"

## 적용 시나리오

- 보유/관심 종목의 주가 급등/급락 (일반적 트리거 기준: 하루 ±5%, 일주일 ±10%)
- 실적 발표 후 주가 이상 변동, 시장이 무엇에 반응하는지 빠르게 파악하고 싶을 때
- 뉴스 헤드라인을 봤는데 소음인지 진짜 신호인지 불확실할 때
- **부적합**: 완전한 투자 리서치(`/investment-team` 사용), 실적 정독(`/earnings-review` 사용), 장기 논지 추적(`/thesis-tracker` 사용)

## 실행 절차

### 1단계: 파라미터와 시나리오 확인

아래 정보를 사용자에게 확인한다 ($ARGUMENTS 에 없는 경우):

| 파라미터 | 설명 | 기본값 |
|------|------|------|
| **회사명** | 한국어/영어/종목코드 모두 가능 | 필수 |
| **시간 창** | 뉴스 정찰 소급 일수 | 기본 14일, 실적 시즌엔 7일로 단축 가능 |
| **주가 변동** | 등락폭 + 기간, 예: "3일간 -12%" | 선택. 있으면 원인 규명에 집중 |
| **관심 초점** | 기업 이벤트 / 규제 / 산업 / 심리 | 기본은 4개 균등 |

사용자가 회사명만 말했으면 먼저 되묻는다: "최근 며칠간의 뉴스인가요? 설명이 필요한 구체적 주가 변동이 있나요?" — **말없이 가정하지 않는다**.

### 2단계: 정보 가용성 등급

`investment-team.md`의 A/B/C 등급을 참조하되 차원이 다르다:

| 등급 | 특징 | 정찰 전략 |
|------|------|---------|
| **A등급 (정보 풍부)** | 대형주, 미디어 커버리지 넓음, 실적 시즌 | 핵심은 **노이즈 제거와 원인 규명** — 정보가 너무 많으면 오히려 진짜 원인을 찾기 어렵다. 각 Agent는 판단력을 갖고 "반복 재전달"된 2차 뉴스를 걸러야 한다 |
| **B등급 (정보 보통)** | 중소형주, 커버리지 보통 | 표준 모드, 핵심 이벤트마다 독립 출처 1-2개 첨부 |
| **C등급 (정보 희소)** | 소형주, 신규 상장, 비주류 | "탐색 모드"로 전환 — 이상 변동을 설명할 뉴스를 전혀 못 찾을 수도 있는데, **그 결론 자체가 가치 있다** (펀더멘털이 아니라 기술적/수급 요인일 가능성) |

등급을 각 Agent에게 알려 정찰 방식에 반영시킨다.

### 3단계: 팀 생성

TeamCreate로 팀을 만든다:
- `team_name`: `{회사명}-newspulse` (영문 소문자, 예: `pdd-newspulse`)
- `agent_type`: `team-lead`

### 4단계: 4개 정찰 태스크 생성

TaskCreate로 아래 4개 태스크를 만든다:

#### 태스크 1: 기업 이벤트 정찰 (company-event-scout)

- **subject**: `{회사명}의 최근 {N}일간 기업 자체 이벤트 정찰`
- **description**:
  1. **공식 공시**: HKEX/SEC/cninfo/DART 등 규제 공시 플랫폼의 최근 공시
  2. **실적과 가이던스**: 최신 분기/연간 보고서, 실적 예고, 실적 발표회 요점
  3. **경영진 움직임**: 임원 변동, 지분 증감, 자사주 매입, 배당, 주식 보상
  4. **중대 사업 이벤트**: 신제품 출시, M&A/구조조정, 사업 분리, 대형 고객/수주
  5. **자본 운용**: 유상증자, 전환사채, ADR 전환, 상장 이전/상장폐지 논의
  6. **소송과 컴플라이언스**: 피소, 자진 공시된 컴플라이언스 이벤트
  7. 각 이벤트에 표기: **날짜 / 출처 링크 / 한 문장 요약 / 주가 변동과의 관련 가능성 (상/중/하)**
  8. 시간 역순의 타임라인 표로 출력

#### 태스크 2: 규제와 정책 (regulatory-watcher)

- **subject**: `{산업/회사}의 최근 {N}일간 규제·정책 변화 정찰`
- **description**:
  1. **산업 규제**: 해당 산업의 신규 규정, 과징금, 시정 조치, 라이선스 변화
  2. **국경 간 정책**: 미중 관계(중국 관련주), 관세, 수출 통제, 데이터 보안
  3. **세금 정책**: 부가가치세, 법인세, 소득세 관련 변화
  4. **반독점과 경쟁법**: 조사, 과징금, 인수 불허
  5. **특수 산업 정책**: 의약품 공동구매, 교육 규제, 부동산 규제, 인터넷 플랫폼 규제 등
  6. **통화와 외환**: 해당 회사에 영향을 주는 환율/금리/자본 통제 변화
  7. 각 정책에 표기: **날짜 / 출처 / 해당 회사에 대한 직접 영향 정도 (직접/간접/무관)**
  8. 핵심 판단: 방금 착지한 "정책 블랙스완"이 있는가

#### 태스크 3: 산업과 경쟁사 (industry-peer-analyst)

- **subject**: `{회사명}의 산업 구도와 경쟁사의 최근 {N}일간 동향 정찰`
- **description**:
  1. **직접 경쟁사**: 핵심 경쟁사 3-5개를 나열하고 각각의 최근 이벤트 확인 (실적, 제품, 가격 전쟁, 인사)
  2. **밸류체인 상하류**: 상류 원자재/공급사, 하류 고객/유통망의 최근 가격, 생산능력, 수주 변화
  3. **산업 전반**: 산업 경기 데이터, 출하량, 수요 측 신호 (소비 데이터, 입찰 데이터 등)
  4. **대체재 위협**: 신기술, 새 업태가 해당 산업에 주는 충격
  5. **산업 지수 퍼포먼스**: 동종 산업 주식의 최근 흐름, 이 회사가 아웃퍼폼/언더퍼폼/동조인지
  6. 핵심 판단: **이것은 회사 개별 이벤트인가, 산업 전체의 베타 변동인가?**
  7. 각 이벤트에 출처와 날짜 표기

#### 태스크 4: 시장 심리와 기관/인플루언서 (sentiment-tracker)

- **subject**: `{회사명}의 최근 {N}일간 시장 심리와 기관 관점 변화 정찰`
- **description**:
  1. **셀사이드 등급 변동**: 골드만삭스, 모건스탠리 등의 최근 등급/목표가 조정
  2. **기관 보유 변화**: 13F 공시(미국 주식), 홍콩 주식 남향 자금, 외국인 자금 흐름
  3. **공매도 데이터**: 공매도 비율, 새로 발표된 공매도 리포트 (있는 경우)
  4. **유명 투자자 관점**: `python3 ~/ai-berkshire/tools/xueqiu_scraper.py`로 돤융핑 등 유명 투자자의 최근 관련 발언 수집 가능
     - 돤융핑 user_id: `1247347556`
     - 명령 예시: `python3 ~/ai-berkshire/tools/xueqiu_scraper.py --user-id 1247347556 --keywords {회사명},{종목코드} --output /tmp/dyp-{회사명}.md`
     - 해당 회사가 돤융핑/리루의 관심 종목일 때만 호출, 아니면 건너뛰어 시간 절약
  5. **루머와 소문**: 미디어가 확인하지 못한 루머, 소셜미디어 토론 이슈 (X/Reddit 등)
  6. **기술적 신호**: 핵심 지지선/저항선 도달 여부, 대량 매매 여부, 신용거래 이상
  7. 핵심 판단: **펀더멘털 주도인가, 심리/수급 주도인가?**

### 5단계: 4개 Agent 병렬 실행

**반드시 같은 메시지에서 Task 도구를 4번 병렬 호출한다**. 각 Agent 설정:
- `subagent_type`: `general-purpose`
- `run_in_background`: `true`
- `team_name`: `{회사명}-newspulse`
- `name`: 해당 역할명 (company-event-scout / regulatory-watcher / industry-peer-analyst / sentiment-tracker)

각 Agent의 prompt 템플릿:

```
당신은 {회사명} 뉴스 펄스 팀의 "{역할명}"으로, {정찰 차원} 차원의 최근 {N}일간 이벤트를 정찰한다.

시간 창: {시작 날짜} ~ {오늘 날짜}
주가 변동 배경: {사용자가 제공한 변동 정보, 없으면 "특정 변동 없음, 정기 점검"이라고 기재}
정보 가용성 등급: {A/B/C}

태스크 #{태스크 번호}를 완료하라: {태스크 subject}

구체적 정찰 요구사항:
{태스크 description의 내용}

**정찰 방법**:
- WebSearch로 시의성 있는 검색을 우선 사용 (키워드에 날짜나 "최근", "latest", "2026" 추가)
- 핵심 이벤트는 WebFetch로 원본 출처를 정독 (공시 원문, 실적 보고서, 규제 문서)
- 각 이벤트에 "독립 출처 검증" 수행 — 루머는 최소 2개의 독립 출처 필요
- **낚시성 제목에 속지 말 것**: 제목과 본문이 다른 이벤트는 "제목 오도" 표기

**출력 형식 (중요)**:
1. **핵심 발견**: 가장 중요한 이벤트 3-5개, 각각 1-2문장
2. **완전한 이벤트 타임라인 표** (날짜 역순):
   | 날짜 | 이벤트 | 출처 | 주가 변동 관련성 | 지속성 |
3. **이 차원의 원인 규명 결론**: 정찰한 이벤트를 근거로 "이 차원이 주가 변동을 설명할 수 있는가? 신뢰도는?"에 답한다
4. **데이터 공백 선언**: 어떤 정보를 못 찾았고, 어디에 의문점이 있으며, 무엇이 추가 정보를 기다려야 하는지
5. "사실"과 "추측"을 엄격히 구분하고 CLAUDE.md의 객관성 원칙을 준수

**완료 후**:
1. TaskUpdate로 태스크를 completed로 표시
2. SendMessage로 완전한 정찰 보고서를 team-lead에게 전송 (type: "message", recipient: "team-lead")
```

### 6단계: 실시간 진행 추적

- 정찰 보고서를 하나 받을 때마다 그 차원의 핵심 발견 3개를 사용자에게 보여준다
- 4개가 모두 도착할 때까지 대기
- 전부 도착하면 SendMessage로 4개 Agent에게 shutdown_request 전송

### 7단계: team-lead 종합 원인 규명

4개 정찰 보고서를 취합해 **이상 변동 원인 규명 보고서**를 출력한다 (리서치 보고서가 아니라 "판단"이 핵심):

---

#### 1. 한 문장 원인 규명
> 한 단락(30-60자)으로: 이번 주가 변동의 주원인 + 부차 원인 + 성격 판단 (가치 이벤트/심리 변동/불명)

#### 2. 완전한 이벤트 타임라인 (4개 차원 통합)

날짜 역순으로 모든 차원의 이벤트를 통합:

| 날짜 | 차원 | 이벤트 | 출처 | 변동 기여 가중치 |
|------|------|------|------|-----------|
| 2026-04-30 | 기업 | XX | 링크 | 🔴 상 |
| 2026-04-29 | 산업 | XX | 링크 | 🟡 중 |
| 2026-04-28 | 심리 | XX | 링크 | ⚪ 하 |

가중치 범례: 🔴 상 (단독으로 변동을 설명 가능) / 🟡 중 (일부 기여) / ⚪ 하 (배경 소음)

#### 3. 원인 규명 표

| 후보 설명 | 증거 | 반증 | 신뢰도 | 지속성 |
|---------|------|------|------|--------|
| 예: 실적 miss | 매출이 예상 대비 5% 하회, 매출총이익률 하락 | 일회성 요인이며 경영진이 설명함 | 상 | 단기 1-2주 |
| 예: 산업 베타 | 동종업계도 같은 기간 8% 하락 | 이 종목의 낙폭이 산업보다 뚜렷이 큼 | 중 | 산업과 동조 |

#### 4. 성격 판단 (핵심 결론)

하나에 체크:

- [ ] **가치 이벤트**: 펀더멘털에 실질적 변화 발생 (실적, 해자, 경영진, 최종 국면). 투자 논지 재심사 필요
- [ ] **심리/기술적 변동**: 펀더멘털 변화 없음. 수급/심리/베타 주도. 기회 또는 소음으로 볼 수 있음
- [ ] **진짜 원인 불명**: 주가 변동 폭에 걸맞은 이벤트를 못 찾음 — **가장 위험한 결론이다**. 시장이 뭔가를 미리 알았거나(내부정보/선행매매), 우리가 정보원을 놓쳤거나
- [ ] **혼합**: 일부 가치 이벤트 + 일부 심리 증폭

#### 5. 차원별 정찰 요약

차원마다 가장 중요한 발견 3-5개 + 해당 차원의 원인 기여도.

#### 6. 행동 제안

| 행동 | 권장 여부 | 이유 |
|------|--------|------|
| 투자 논지 재심사 트리거 (`/thesis-tracker`) | | |
| 심층 실적 정독 트리거 (`/earnings-review`) | | |
| 경영진 재평가 트리거 (`/management-deep-dive`) | | |
| 리밸런싱 (비중 확대/축소/유지) | | 참고 제시일 뿐, 최종 결정권은 사용자에게 |
| 관찰만 | | |

#### 7. 향후 7-30일 추적 목록

- [ ] 공시 예정 이벤트 1 (예: 5/15 실적 발표회)
- [ ] 추적할 지표 2
- [ ] 핵심 관찰 신호 3

#### 8. 정보 공백 선언

이번 정찰에서 해결하지 못한 의문점, 찾지 못한 정보, 추가 공시를 기다려야 하는 사항을 정직하게 나열한다. **추측으로 채우느니 "불확실"이라고 표기하는 편이 낫다**.

### 8단계: 보고서 저장

`reports/{회사명}/{회사명}-news-{YYYYMMDD}.md`에 저장한다. `reports/{회사명}/` 디렉토리가 없으면 생성한다 (그 회사의 리서치 보고서가 아직 없다는 뜻).

### 9단계: 팀 정리

TeamDelete로 팀 리소스를 정리한다.

## 핵심 원칙

1. **빠름이 완전함을 이긴다** — 이 skill의 핵심 가치는 10-15분 안에 원인 판단을 내는 것이다. 심층 분석에 빠지지 마라 (그건 다른 skill의 몫)
2. **나열보다 원인 규명** — 이벤트를 찾는 건 어렵지 않다. 어려운 것은 "어떤 이벤트가 이번 주가 변동에 걸맞은가"의 판단이다. 덧셈이 아니라 뺄셈을 하라
3. **"불명"을 정직하게 마주하라** — 주원인을 못 찾으면 "진짜 원인 불명"이라고 명확히 쓴다. 억지로 인과 사슬을 꿰맞추는 것보다 가치 있다 (시장이 악재를 선반영 중일 수 있다)
4. **입장을 미리 정하지 않는다** — 보유 중이라고 해서 "심리적 변동일 뿐, 괜찮다" 쪽으로 기울지 마라. 증거가 가리키는 쪽으로 쓴다
5. **"촉매"와 "우연"을 구분하라** — 같은 시기에 일어난 이벤트가 반드시 변동의 주원인은 아니다. 영향의 규모가 걸맞은지 봐야 한다
6. **정보 가용성을 존중하라** — C등급 회사는 정말 뉴스를 하나도 못 찾을 수 있다. 그 결론 자체를 써야 한다
7. **`CLAUDE.md` 객관성 원칙 준수** — 모든 판단에 데이터 출처를 붙이고 사실과 관점을 구분한다
8. **사용자 대신 결정하지 않는다** — 원인 규명과 행동 제안 목록을 주되, 매매 결정은 사용자가 한다
