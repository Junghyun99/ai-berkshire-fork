# 비상장 기업 연구: 멀티 Agent 병렬 심층 리서치 프레임워크

$ARGUMENTS 에 대해 팀 기반 심층 리서치 분석을 수행한다. 앤트그룹, 샤오홍슈, SpaceX, Stripe 같은 비상장 기업 전용으로 설계되었다.

**최종 목표**: 정보가 태생적으로 희소한 조건에서 이 회사의 **진짜 가치**를 최대한 복원한다 — 시장이 매긴 밸류에이션이 아니라 사업 자체가 얼마짜리인지.

## 프레임워크 특징

비상장 기업 vs 상장 기업 리서치의 핵심 차이:
- **표준화된 재무제표 없음**: 다중 출처 짜맞추기와 교차 검증 필요
- **밸류에이션 앵커 부족**: 투자 라운드, 유사기업 비교법, 시나리오 추론에 의존
- **정보 비대칭 큼**: "퍼즐 맞추기식" 리서치 방법이 더 많이 필요
- **엑싯 경로 불확실**: IPO/피인수/구주 거래 모두 가능

## AI 리서치 편향 자각 (이 프레임워크의 핵심 전제)

비상장 기업은 AI 리서치 편향이 가장 심한 영역이다. 아래 함정을 항상 경계해야 한다:

**핵심 모순**: AI는 이미 있는 정보의 구조화에 강하지만, 비상장 기업의 정보는 태생적으로 희소하다. 그 결과:
1. **가짜 보수성**: 자료가 적으니 AI는 보수적/모호한 결론으로 기울지만, 자료 부족 ≠ 나쁜 회사
2. **가짜 정밀함**: 보고서 템플릿을 채우려고 AI가 "합리적 추측"을 "근거 있는 분석"으로 위장할 수 있음
3. **벤치마킹 함정**: 상장사와 억지로 비교하면 상장사의 밸류에이션 논리를 물려받아 비상장사의 고유 가치를 놓침
4. **생존자 편향**: 인터넷에서 검색되는 회사 정보는 대개 긍정 편향 (회사가 능동적으로 퍼뜨리는 것은 대부분 좋은 소식)

**대응 원칙**:
- 추측으로 표를 채워 확실성을 가장하느니 여백을 남기고 "모른다"고 말한다
- 모든 데이터 포인트에 신뢰도(🟢높음/🟡중간/🔴낮음)를 표기해 독자가 스스로 판단하게 한다
- "검증 가능한 사실"과 "AI의 추론"을 구분해 다른 형식으로 표기한다
- 정보가 극도로 희소한 회사는 "제1원리 모드"로 전환한다 — 보고서 완결성을 추구하지 말고 핵심 질문 몇 개만 답한다:
  1. 이 사업은 어떤 진짜 문제를 푸는가? 수요가 진짜인가 가짜인가?
  2. 왜 이 팀인가? 그들의 고유한 우위는 무엇인가?
  3. 성공하면 천장이 얼마나 높은가? 실패하면 어디서 죽을 가능성이 가장 큰가?
  4. 현 단계의 핵심 검증 노드는 무엇인가?

**정보 비대칭의 역이용**: 시장이 비상장 기업의 정보를 적게 안다 → 가격 결정 효율이 낮다 → 바로 그것이 초과수익의 원천일 수 있다. AI 리서치의 목표는 정보 비대칭의 제거(불가능)가 아니라, 제한된 정보에서 가장 핵심적인 판단 근거를 추출하는 것이다.

### 1단계: 팀 프레임워크 제시

사용자에게 아래 팀 구조를 보여주고, 확인 후 시작한다:

| 역할 | 책임 | 핵심 관점 |
|------|------|----------|
| **team-lead** (본인) | 총괄 조율, 정보 퍼즐, 교차 검증, 최종 보고서 출력 | 투자 의사결정 통합 |
| **business-decoder** | 비즈니스 모델 분해 & 제품·사용자 분석 | "이 사업의 본질은 무엇인가" |
| **financial-detective** | 재무 데이터 짜맞추기 & 밸류에이션 추론 | "정보가 없는 조건에서 진짜 재무 모습을 최대한 복원" |
| **competitive-mapper** | 산업 구도 & 경쟁 상황 & 대체 위협 | "누가 경쟁하고, 누가 뒤집을 수 있나" |
| **risk-governance-analyst** | 리스크 전경 & 경영진/지배구조/투자자 평가 | "무엇이 잘못될 수 있고, 누가 키를 잡고 있나" |
| **tech-ip-analyst** | 기술 스택/특허/R&D 역량/기술 해자 | "기술 장벽이 진짜인가 가짜인가, 얼마나 버티나" |
| **signal-miner** | 대체 데이터 발굴: 채용/특허/소송/앱 데이터/공급망 | "일반 정보 밖에 어떤 단서가 더 있나" |

## 재무 데이터 짜맞추기 (탐정식 리서치)

### 2. 핵심 재무 지표 추산
아래 데이터를 최대한 추산하되, **모든 데이터 포인트에 반드시 명기**: 출처, 시점, 신뢰도, 추산 방법.

**매출 측**:
- 총매출 규모와 성장률 (최근 3년, 연간/분기 단위)
- 매출 구조 분해 (사업 라인/제품 라인별)
- 매출 성장 동인 분해: 양(사용자/거래량) × 가격(ARPU/객단가)
- 매출의 계절적 변동 패턴

**비용 측**:
- 매출총이익률 추산 (동종 상장사 벤치마킹, 선정 논리 설명)
- R&D 비용률 추산 (직원 수 × 인당 급여, 또는 R&D 인력 비중으로 추산)
- 판매비용률 추산 (고객 획득 채널과 광고 집행 데이터로 추산)
- 관리비용률 추산

**이익 측**:
- 영업이익/EBITDA 추산
- 순이익/조정 순이익 추산
- 흑자 타임라인: 언제 흑자? 아직 적자면 흑자 예상 시기는?

**현금흐름 측**:
- 영업현금흐름 판단 (플러스/마이너스, 자체 현금 창출 여부)
- 자본지출 수준과 추세
- 잉여현금흐름 추산
- 보유 현금/자금 소진 속도 (투자 유치 금액, 유치 간격, 직원 수로 추산)
- 현금 활주로: 현재 소진 속도로 얼마나 버티나?

**효율 지표**:
- 직원 수와 인당 효율 (인당 매출, 인당 이익)
- 자본 효율: 1원을 조달할 때마다 매출이 얼마나 생기나
- 동종 상장사 효율 지표와 비교

### 4. 투자 유치 이력과 밸류에이션 변화
완전한 투자 유치 타임라인 정리:

| 라운드 | 시기 | 금액 | 프리머니 | 포스트머니 | 리드 투자자 | 팔로우 투자자 | 밸류에이션 배수 성장 | 비고 |
|------|------|------|---------|---------|--------|--------|------------|------|

분석:
- 밸류에이션 성장 곡선이 건강한가 (라운드별 성장 배수가 합리적인가)
- 유치 간격이 합리적인가 (너무 잦다=소진 빠름? 너무 뜸하다=유치 곤란?)
- 다운 라운드(Down Round)가 있었는가
- 기존 주주가 계속 추가 투자하는가 (신뢰 신호)
- 최근 라운드의 투자 조건 추정:
  - 우선청산권 (1x/2x/participating)
  - 희석 방지 (full ratchet/weighted average)
  - 조건부 계약 (실적/상장 기한)
  - 이 조건들이 보통주 가치에 미치는 영향

### 5. 밸류에이션 분석 (다중 방법 교차)

**방법 1: 최근 투자 라운드 밸류에이션법**
- 최근 라운드의 밸류에이션과 시점
- 우선청산권 등 조건을 감안한 "보통주 등가 밸류에이션" (통상 20-40% 할인 필요)
- 시간 경과에 따른 조정
- 이 라운드 투자자의 동기 분석 (재무적 투자 vs 전략적 투자, 전략 투자자는 프리미엄을 냈을 수 있음)

**방법 2: 유사 상장사 비교법**
- 상장 유사 기업 3-5개 선정, 선정 이유 설명
- 핵심 배수 비교:

| 유사 기업 | P/S | P/E | EV/EBITDA | EV/Revenue | 성장률 | 이익률 |
|---------|-----|-----|-----------|------------|------|--------|

- 적용할 조정:
  - 유동성 할인: 20-30% (구체적 수치와 이유 표기)
  - 성장률 프리미엄/디스카운트
  - 규모 디스카운트
  - 규제/정책 리스크 디스카운트

**방법 3: DCF 시나리오 분석**
세 가지 시나리오, 각각 핵심 가정을 나열한다:

| 최종 상태 영업이익률 | | | |
| 최종 상태 성장률 | | | |
| WACC | | | |
| 터미널 배수(EV/EBITDA) | | | |

모든 가정에는 논거가 있어야 하고, 허공에서 가정하면 안 된다.

**방법 4: 최종 국면 시총 역산법**
- 5년/10년 후 이 사업이 최종 상태에서 차지할 시장 지위를 가정
- 최종 상태의 매출과 이익률 가정
- 최종 상태의 합리적 배수 (동종 산업 성숙 기업 참조)
- 현재의 합리적 밸류에이션 구간 역산
- 내포된 연환산 수익률

**방법 5: 거래 벤치마킹법**
- 동종 산업 최근 2년의 M&A/투자 거래
- 거래 배수 (P/S, P/E)
- 거래 배경과 프리미엄/디스카운트 요인

### 4단계: 6개 Agent 병렬 실행

`Agent` 도구로 6개 서브에이전트를 동시에 실행한다 (**반드시 같은 메시지에서 6번 병렬 호출**):

각 `Agent` 호출 설정:
- `subagent_type`: 해당 역할 (`business-decoder` / `financial-detective` / `competitive-mapper` / `risk-governance-analyst` / `tech-ip-analyst` / `signal-miner`)
- `run_in_background`: `true`
- `description`: 짧은 과제명 (예: `앤트그룹 재무 짜맞추기`)
- `prompt`: 아래 템플릿

각 서브에이전트는 비상장 기업 연구의 분석 프레임·데이터 규율(신뢰도 표기, 사실/추론 구분)을 `.claude/agents/{역할}.md`에 이미 갖고 있으므로, `prompt`에는 구체적 과제만 담는다:

```
{회사명}(비상장 기업)에 대해 다음 리서치 과제를 완료하라: {해당 과제 subject}

구체적 요구사항:
{해당 과제 description의 내용}

**리서치 방법**:
1. WebSearch로 최신 공개 정보를 검색, 차원마다 최소 3-5회, 다른 키워드 조합 사용
2. 검색 키워드 전략:
   - 한국어/현지어: 회사명+매출/밸류에이션/투자유치/사용자수/MAU/IPO/상장신청/구조조정
   - 영어: Company Name + revenue/valuation/funding/users/IPO/filing
   - 구체적 인물명+회사명 (경영진 관련 정보 검색)
   - 회사명+구체적 경쟁사명 (경쟁 동향 검색)
3. 중점 정보원:
   - 높은 신뢰도: 상장 신청서, 규제 문서, 상장사 연차보고서의 관계사 공시
   - 중간 신뢰도: The Information, LatePost, Bloomberg, Reuters, TechCrunch
   - 보조 검증: 커뮤니티, Glassdoor, 기업 정보 조회 플랫폼
4. WebFetch로 핵심 기사의 전문을 가져온다 (검색 요약만 보지 말 것)
5. 중요 데이터는 최소 2개의 다른 출처로 교차 검증

**데이터 표기 규범 (엄격 준수)**:
- 모든 핵심 데이터에 출처 표기 (매체명과 기사 제목까지 구체적으로)
- 데이터 시점 표기 (연월까지 정확히)
- 신뢰도 표기: 🟢높음(상장신청서/공식 공시) / 🟡중간(신뢰할 만한 매체/리포트) / 🔴낮음(추산/소문)
- 출처 간 데이터가 충돌하면 **전부 나열**하고 차이와 당신의 판단을 설명
- "사실"과 "추론"을 구분: 사실은 일반 서체, 추론/추산은 *이탤릭*으로 표기하고 추산 방법 명기
- 확보할 수 없는 정보는 "데이터 누락"을 명확히 표기하고 지어내지 않는다

**출력 요구사항**:
- 보고서는 상세하게, 핵심 데이터는 Markdown 표로 제시
- 각 분석 차원마다 명확한 결론과 점수
- 추산 과정은 완전히 투명하게 (계산 논리와 각 단계의 가정 제시)
- 보고서 말미에 제공:
  1. 이 차원의 총평 점수 (★1-5)와 핵심 판단
  2. 이 차원의 정보 완전성 자가평가 (충분/보통/부족/심각 부족)
  3. 가장 중요한 발견 3가지
  4. 가장 큰 정보 사각지대 (어떤 정보의 부재가 판단에 가장 영향을 주는가)
```

### 5단계: 보고서 수신 및 진행 추적

- 사용자에게 진행 표를 실시간으로 보여준다 (어느 Agent가 완료됐고 어느 Agent가 아직 연구 중인지)
- 보고서를 하나 받을 때마다 진행을 갱신하고 그 보고서의 핵심 요점(3-5개)을 보여준다
- 6개 보고서가 모두 도착할 때까지 대기

### 6단계: 교차 검증과 정보 퍼즐

**강화판 프레임워크에서 가장 핵심적인 추가 단계다.** 취합 전에 team-lead는:

1. **데이터 충돌 중재**:
   - 각 Agent 보고서의 핵심 데이터 추출
   - 서로 다른 Agent가 인용한 같은 데이터가 일치하는지 식별
   - 충돌 데이터를 중재: 모든 출처를 나열하고 무엇을 채택했는지, 왜인지 설명

2. **신호 일관성 검증**:
   - 사업 성장 신호(business-decoder) vs 채용 신호(signal-miner)가 일치하는가?
     (사업이 빠르게 성장한다는데 채용이 축소 중이면 설명이 필요하다)
   - 기술 선도 내러티브(tech-ip) vs 특허/인재 데이터(signal-miner)가 뒷받침하는가?
   - 밸류에이션 수준(financial-detective) vs 경쟁 지위(competitive-mapper)가 맞는가?
   - 경영진의 공개 내러티브(risk-governance) vs 실제 행동 신호(signal-miner)가 일치하는가?

3. **정보 퍼즐 복원**:
   - 6개 보고서의 정보 조각을 맞춰 더 완전한 그림을 복원할 수 있는지 확인
   - 정보의 "백색 지대"(확인된 사실), "회색 지대"(단서는 있으나 불확실), "흑색 지대"(완전히 미지) 표기

4. **반편향 점검**:
   - 보고서에 "긍정 정보는 상세하고 부정 정보는 간략한" 편향이 있는지 확인
   - 모든 긍정 판단에 대응하는 반대편 검증이 있는지 확인

## 중요 주의사항

1. **6개 Agent는 반드시 병렬 실행** — 같은 메시지에서 `Agent` 도구를 6번 호출한다. 서브에이전트의 최종 메시지가 team-lead에게 툴 결과로 반환되며, `SendMessage`/`TeamCreate`/`TeamDelete`는 이 하네스에 없으므로 쓰지 않는다
2. **데이터 신뢰도 표기** — 비상장 기업의 데이터 출처는 들쭉날쭉하므로 모든 핵심 데이터에 출처와 신뢰도를 표기해야 한다
3. **추산은 투명하게** — 모든 추산 과정은 계산 논리를 보여야 하며, 허공에서 숫자를 주면 안 된다
4. **교차 검증** — 핵심 데이터는 최소 2개 출처로 교차 검증, 출처가 충돌하면 전부 나열
5. **신호 일관성 검증** — 취합 단계에서 반드시 차원 간 신호 일관성 점검을 수행
6. **결론은 명확하게** — 투자/관망/회피 제안을 회피하지 않되, 동시에 결론의 신뢰도를 설명
7. **인내심 있게 대기** — 6개 Agent의 리서치에는 몇 분이 걸린다. 사용자에게 진행 상황을 실시간 갱신
8. **다국어 검색** — 비상장 기업의 정보는 여러 언어의 매체에 분산되어 있을 수 있으니 여러 언어로 검색해야 한다
9. **반편향 핵심 원칙** — 자료 부족 ≠ 나쁜 회사, AI 분석 분량 짧음 ≠ 투자 확실성 낮음. 정보가 극도로 희소한 회사는 "제1원리 모드"로 전환해 핵심 질문에 집중하고 보고서 형식의 완결성을 추구하지 않는다
10. **정직한 여백** — 보고서에서 "근거 있는 분석"과 "추측 채우기"를 명확히 구분하고, "이 차원은 데이터가 부족해 의미 있는 결론을 낼 수 없음"의 등장을 허용한다
11. **대체 데이터는 소음이 아니다** — 채용, 특허, 소송, 앱 데이터 등 대체 데이터가 뉴스 보도보다 실제 경영 상황에 더 가까울 수 있다
12. **진짜 가치 지향** — 최종 목표는 이 사업이 얼마짜리인지 판단하는 것이지, 보기 좋은 보고서를 내는 것이 아니다. 신뢰할 만한 밸류에이션 판단에 정보가 부족하면 "정보 부족, 신뢰할 만한 밸류에이션 불가"라고 바로 말한다
