# 버핏 가치투자 매수 전 Checklist

$ARGUMENTS 에 대해 버핏 가치투자 매수 전 Checklist 분석을 수행한다.

**지원하는 입력 형식**: 단일 또는 복수 회사, 쉼표/공백으로 구분. 예: `텐센트, 마오타이, 엔비디아` 또는 `NVDA AAPL MSFT`

## 실행 절차

### 1단계: 입력 해석, 분석 대상 회사 전부 식별

$ARGUMENTS 에서 모든 회사명/코드를 해석한다. 각 회사에 대해 확정한다:
- 회사 정식 명칭, 종목코드, 상장 거래소
- 비상장 회사면 "비상장"으로 표기하고 간단히 설명 (간접 투자 경로가 있는지). 전체 Checklist는 생략

### 1.5단계: AI 리서치 편향 사전 경고

각 회사에 대해 "정보 풍부도"를 빠르게 등급화(A/B/C)하고 보고서에 표기한다:

| 등급 | 판단 기준 | Checklist에 미치는 영향 |
|------|---------|-----------------|
| A등급 | 상장 오래됨, 데이터 풍부 | 정상 실행하되 "컨센서스 함정" 경계 — 모든 지표가 명확해 보인다고 정말 확실한 것은 아니다 |
| B등급 | 데이터 제한적, 추산 필요 | 추산 지표마다 신뢰도 표기, "좋은 사업" 판단 시 데이터 신뢰성을 가중 고려 |
| C등급 | 정보 극도로 희소 | 6관문 표를 억지로 채우지 말고 "데이터 부족으로 판단 불가"를 정직하게 표기, 검증 가능한 핵심 질문에 집중 |

**핵심 원칙**: Checklist의 목표는 **나쁜 선택의 배제**다. C등급 회사에서 "데이터 부족"은 "미통과"도 "통과"도 아니다 — AI가 표를 못 채운다고 부결로 판정하지 말고 "회색 지대, 1차 정보 보완 필요"로 정직하게 표기해야 한다.

돤융핑의 말: "이해 못한다"에는 두 가지가 있다 — 사업이 너무 복잡해서 정말 이해 못하는 것과, 아직 시간을 들여 보지 않은 것. AI 리서치의 한계는 "자료가 적음"과 "이해 못함"을 혼동하기 쉽다는 것이다.

### 2단계: 병렬 데이터 수집

Task 도구로 **회사마다** 독립 백그라운드 Agent를 띄워 데이터를 수집한다 (모든 회사 동시 병렬 실행). 각 Agent의 수집 항목:

1. **수익성**: ROE (5-10년 추세), 매출총이익률, 순이익률, 잉여현금흐름
2. **밸류에이션 데이터**: 현재 주가, 시가총액, PER(TTM), 선행 PER, PBR, 배당수익률
3. **성장 추세**: 최근 3년 매출/이익 성장률
4. **재무 건강**: 부채 수준, 자본지출 수요, 현금 보유, 순현금/순부채
5. **경쟁 구도**: 시장 점유율, 주요 경쟁사, 점유율 변화 추세
6. **해자 증거**: 브랜드/전환비용/네트워크 효과/규모의 경제/기술 장벽의 구체적 증거
7. **경영진 기록**: CEO 이력, 핵심 결정, 지분, 자본 배분 기록
8. **최신 동향**: 최근 6개월 중대 이벤트 (실적, M&A, 규제, 경영진 변동 등)

### 3단계: 회사별 6관문 Checklist 실행

상장된 각 회사에 대해 차례로 6개 관문을 통과시킨다:

---

#### 제1관문: 나는 이 사업을 이해하는가 (능력의 범위)

반드시 답할 것:
- [ ] 이 회사가 무엇으로 돈을 버는지 한 문장으로 말할 수 있는가?
- [ ] 10년 후에도 높은 확률로 무슨 사업을 하고 있을까?
- [ ] 어떤 핵심 변수가 성패를 좌우하는가?
- [ ] 이 산업에 대한 인식은 깊은 리서치에서 왔는가, 주워들은 것인가?

**채점 기준** (★1-5):
- ★★★★★: 비즈니스 모델이 지극히 단순 명확, 10년 확실성 높음 (예: 마오타이 — 술 빚어 술 판다)
- ★★★★☆: 모델은 명확하나 기술 장벽이 있어 일정한 전문 지식 필요
- ★★★☆☆: 모델은 이해 가능하나 10년 확실성이 낮고 산업 변화 빠름
- ★★☆☆☆: 사업 라인이 복잡하거나 산업이 격변 중이라 미래 예측 어려움
- ★☆☆☆☆: 완전히 능력 범위 밖

**하드 부결**: 돈 버는 방식조차 설명 못하면 바로 "능력 범위 밖, 분석하지 않음"으로 표기.

---

#### 제2관문: 이것은 좋은 사업인가 (경제적 특성)

데이터로 말한다. **핵심 지표는 반드시 도구로 정밀 계산**:

```bash
python3 tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당순자산} --fcf-per-share {주당 FCF} --dividend {주당 배당}
```

| 지표 | 이 회사의 수치 | 참고 기준 | 판단 |
|------|-----------|---------|------|
| ROE (5년 평균) | | >15% 우수, >20% 탁월 | |
| 매출총이익률 | | >40%면 가격결정력 시사 | |
| 잉여현금흐름 | | 지속 플러스, ≈순이익 | |
| 자본지출 강도 | | 경자산이 중자산보다 우위 | |
| 부채 수준 | | 이자부 부채/순이익 <3년 | |

**채점 기준** (★1-5):
- ★★★★★: ROE>25%, 고마진, 강한 FCF, 경자산, 저부채 (전부 충족)
- ★★★★☆: 4개 충족
- ★★★☆☆: 3개 충족
- ★★☆☆☆: 2개 충족 또는 추세 악화
- ★☆☆☆☆: 대부분 미달 또는 FCF 지속 마이너스

---

#### 제3관문: 해자가 충분히 깊은가 (경쟁 우위)

항목별 점검:

| 해자 유형 | 보유 여부 | 구체적 증거 | 넓어지는가 좁아지는가? |
|-----------|---------|---------|--------------|
| 브랜드/가격결정력 | | | |
| 전환 비용 | | | |
| 네트워크 효과 | | | |
| 비용/규모 우위 | | | |
| 기술/특허 장벽 | | | |

추가 검증: 경쟁사에게 100억을 주면 이 사업을 복제할 수 있는가?

**채점 기준** (★1-5):
- ★★★★★: 다중 해자가 중첩되고 넓어지는 중
- ★★★★☆: 최소 하나의 강한 해자가 있고 안정적
- ★★★☆☆: 해자는 있으나 깊이가 부족하거나 추세 불명
- ★★☆☆☆: 해자가 침식되는 중
- ★☆☆☆☆: 뚜렷한 해자 없음

---

#### 제4관문: 경영진을 신뢰할 수 있는가 (사람 요인)

| 점검 항목 | 평가 |
|--------|------|
| 정직성 (약속 vs 이행) | |
| 자본 배분 능력 (자사주 매입/배당/M&A 기록) | |
| 주주 이익 지향 (지분, 보수) | |
| 오너 마인드 (창업자 vs 전문경영인) | |
| 기업 지배구조 (특수관계자 거래, 영업권, 감사) | |
| CEO가 떠나도 정상 운영되는가? | |

**채점 기준** (★1-5):
- ★★★★★: 창업자가 키를 잡고, 자본 배분 탁월, 이해관계 완전 일치
- ★★★★☆: 경영진 우수하나 작은 흠 존재
- ★★★☆☆: 경영진 합격이나 지배구조에 잠재 리스크
- ★★☆☆☆: 정직성 또는 지배구조 문제 존재
- ★☆☆☆☆: 심각한 정직성 문제 (→하드 부결)

#### 제6관문: 비중과 의사결정 규율 (감정 폭주 방지)

아래 감정 신호를 점검한다:
- FOMO 때문에 사고 싶은 것은 아닌가?
- 남이 추천해서 사고 싶은 것은 아닌가?
- 5년간 거래 정지되어도 받아들일 수 있는가?
- 매수 논리를 200자 이내로 명확히 쓸 수 있는가?

### 4단계: 거울 테스트

각 회사에 대해 거울 테스트 문장을 작성한다:

> "나는 ___원에 ___회사를 매수한다. 이유는:
> 1. 이 사업의 본질은 ___이고, 나는 그것을 이해한다;
> 2. 해자는 ___이며, 넓어지고/좁아지고 있다;
> 3. 경영진은 ___하며, 신뢰할 수 있다/없다;
> 4. 현재 가격은 내재가치의 ___수준이고, 안전마진이 충분하다/없다;
> 5. 내가 틀리더라도 하방 리스크는 통제 가능/불가능하다. 왜냐하면 ___."

**5문장을 온전히 못 쓰면 = 사지 않는다.** "통과" 또는 "미통과"를 명확히 표기.

---

### 5단계: 빠른 부결 리스트

각 회사를 항목별로 점검하고, 하나라도 걸리면 바로 "부결"로 표기한다:

- [ ] 이 회사가 어떻게 돈을 버는지 설명할 수 없다
- [ ] 3년 연속 잉여현금흐름 마이너스이고 개선 기미가 없다
- [ ] 경영진에게 정직성 오점이 있다
- [ ] 경쟁 우위가 비가역적으로 침식되고 있다
- [ ] "다음 매수자가 더 높은 가격을 내는 것"에 기대야 돈을 번다 (폭탄 돌리기)
- [ ] 이 투자가 0이 되는 결과를 감당할 수 없다
- [ ] 매수 이유가 주로 "다들 사니까" 또는 "요즘 잘 오르니까"다
- [ ] 매수 이유를 200자 이내로 명확히 쓸 수 없다

---

### 6단계: 총괄 비교표 출력 (복수 회사 시 필수)

여러 회사를 분석할 때는 반드시 비교 총괄표를 생성한다:

| 회사 | Checklist 통과? | 능력범위 | 좋은사업 | 해자 | 경영진 | 안전마진 | 핵심 결론 |
|------|----------------|--------|--------|--------|--------|---------|---------|
| | | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ | |

---

### 7단계: 최종 결론과 파일 저장

각 회사에 명확한 결론을 준다 (회피하지 않는다):
- ✅ **Checklist 통과** (X/6 관문) — 심층 리서치 단계로 진입 가능
- ❌ **Checklist 미통과** — 어떤 레드라인에 걸렸는지 설명
- ❓ **회색 지대** — 핵심 쟁점이 무엇이고, 투자자가 스스로 판단해야 할 것이 무엇인지 설명
- N/A — 비상장/매수 불가

완전한 보고서를 `~/버핏Checklist-[회사명 또는 "다중회사비교"].md`에 저장한다.

## 출력 형식 요구사항

1. 회사마다 독립된 장으로 구성: 6관문 점수표 + 핵심 데이터 표 + 핵심 리스크 (3-5개) + 거울 테스트 + 명확한 결론
2. 복수 회사일 때는 마지막에 총괄 비교표 첨부
3. 모든 점수는 ★기호 사용 (★1-5), 반 별 없음
4. 데이터에는 출처와 시점을 표기하고, 추정값에는 "추정"을 명기
5. 글 말미에 버핏의 명언으로 맺는다: "투자의 제1원칙은 돈을 잃지 않는 것이다"
6. 언어 스타일: 직접적이고 예리하게, 군말 없이. 버핏/멍거/돤융핑의 어록을 곁들여 코멘트

## 핵심 원칙

- **놓칠지언정 틀리지 않는다**: Checklist의 목표는 나쁜 선택의 배제이지, 최고를 찾는 것이 아니다
- **능력 범위를 정직하게 마주한다**: 모르면 모른다고 말하고, 억지로 분석하지 않는다
- **안전마진은 생명선이다**: 좋은 회사도 비싸게 사면 돈을 잃는다
- **거울 테스트는 건너뛸 수 없다**: 이유를 명확히 말할 수 없으면 사지 않는다. 예외 없다
